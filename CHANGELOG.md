## [v1.1.1] — 2026-06-25

### Добавлено

- **Поддержка `.env` файла** — переменные окружения читаются из `.env` рядом с бинарником или `docker-compose.yml`; `BUHGALTER_ALLOWED_HOSTS` для доступа с удалённого сервера без reverse proxy (localhost/127.0.0.1 доступны всегда)
- **Документация OpenAPI без авторизации** — страница `/docs` (Redoc) и `/docs/openapi.yaml` доступны без входа; в README добавлены ссылки на демо-стенд

### Изменено

- **Счета** — пункт «Счета» перенесён из вкладок Настроек в верхнее меню навигации; страница `/accounts` теперь показывает список счетов (раньше редиректила в Настройки)
- **Курсор** — при наведении на все кнопки и кликабельные элементы теперь отображается указатель (рука) вместо стрелки
- **Высота кнопки «Новый счёт»** на главной — приведена к единой высоте с иконкой-кнопкой «+»
- **Расстояние между «Админка» и именем пользователя** в шапке увеличено на 5px

### Исправлено

- Подсветка активного раздела в верхнем меню

### Техническое

- Версия `1.1.1`

## [v1.1.0] — 2026-06-25

### Добавлено

**Сброс пароля (через администратора)**

- Страница входа: «Запросить сброс пароля» → `POST /api/v1/auth/request-password-reset` (при неизвестном логине — тот же успешный ответ, без перечисления учёток)
- Баннер и список ожидающих запросов у администратора; подтверждение → `POST /api/v1/admin/password-reset-requests/{id}/ack`
- Сброс пароля в админке → `PUT /api/v1/admin/users/{id}/password` с инвалидацией сессий пользователя
- Таблица `password_reset_requests` (миграция `023`); rate limit 5 запросов/мин с IP на endpoint сброса

**Кредиты**

- Редактирование сумм неоплаченных платежей → `PATCH /api/v1/credits/{id}/schedule` (только `kind=scheduled`, `is_applied=false`; обновляются связанные `future`-операции)
- При создании кредита задним числом: списание со счёта для хвоста ретро-платежей → поле `retroactive_debit_count` в `POST /api/v1/credits` (расход на `debit_account_id`, учёт в статистике)
- Автодополнение укороченных графиков активных кредитов при старте (`RepairShortSchedules`; миграция-маркер `020`)

**Переводы и категории**

- Поле **комиссии** при переводе; расход в системной категории «Комиссия» (иконка `percent`)
- Системная категория «Комиссия» для новых и существующих пользователей (backfill при старте)
- При drag-order категорий системные остаются в конце; системную категорию нельзя назначить главной

**Интерфейс**

- Мобильная адаптация: touch-targets 44px, карточки графика/операций, bottom sheet модалок, скролл вкладок, меню в шапке с пунктом «Счета»
- Сброс формы при повторном открытии «Новый кредит»
- В админке — блок доната и ссылка на репозиторий GitHub
- Переключатели (`ToggleSwitch`) вместо чекбоксов: кредиты, долги, уведомления, импорт, фильтры, админка
- Редактирование перевода из списка операций (страница счёта)
- Фильтр категорий по типу операции (доход/расход) на `/transactions`, `/stats`, странице счёта
- Суффиксы «(Доход)» / «(Расход)» у категорий с одинаковым именем (например «Долги»)
- Иконка «Архивировать» на странице счёта (как у остальных действий)
- Предупреждение при удалении **оплаченного** платежа по кредиту; подсказка в группе «Оплаченные» графика

**Статистика и фильтры**

- Фильтры по дате **без времени**; границы суток в часовом поясе пользователя (`/stats`, `/transactions`, страница счёта)
- `/stats`: автоприменение фильтров, поиск по описанию операций в блоке drill-down
- Сводка на странице счёта: входящие переводы в «Доходе», исходящие в «Расходе» (согласовано с балансом)

**Документация и демо**

- Демо-стенд: [buhgalter-demo.kai-zer.ru](https://buhgalter-demo.kai-zer.ru/) (логин `demo`, пароль `demo_1_demo`)
- [docs/release-notes-v1.1.md](docs/release-notes-v1.1.md) и обновление справочников в `docs/`
- Roadmap: [roadmap/receipt-scanner.md](roadmap/receipt-scanner.md) — сканер чеков (черновик)

### Изменено

- Генератор графика с процентами: ровно `term_months` платежей; аннуитет 1…N−1, последний — остаток + проценты за период
- Сортировка категорий в API: пользовательские, затем системные (`sort_order`, `name`)
- OpenAPI **1.1.0**: комиссия переводов, редактирование графика, `retroactive_debit_count`, password reset
- Операции с флагом `credit_payment_linked` нельзя менять через `PATCH /transactions/{id}` — только на странице кредита
- Поле суммы (`MoneyInput`): курсор не прыгает в конец при правке в середине числа

### Исправлено

- Укороченный график кредита (например 22 платежа вместо 36) — расчёт и восстановление при старте
- Мобильное меню: перекрывалось контентом (z-index); не закрывалось при навигации
- Форма создания кредита сохраняла данные предыдущего кредита без перезагрузки страницы
- Статистика и список операций расходились по платежам кредита: прошедшие `future`-операции активируются при чтении; в SQL учитываются оплаченные платежи с `exclude_from_stats=0`
- Сводка на странице счёта не учитывала входящие переводы в блоке «Доход»
- Первоначальная настройка (`POST /setup`): повтор после частичного сбоя не давал 500 — восстановление маркера `.configured`, ответ 409 если БД уже настроена
- Календарь (`DateTimePicker`): в конце месяца отображались дни 31–34 вместо дней следующего месяца; дни соседних месяцев в сетке теперь кликабельны
- Тёмная тема: календарь, выпадающие списки и мобильное меню сливались с фоном (`popover-panel`)
- Долги: дата операции и плановый возврат — только дата (без времени в форме и списках)
- `npm run check` / `vite build` не обращались к API за `allowedHosts` (загрузка только при `vite dev`)

### Техническое

- Миграции: `020_repair_credit_schedules`, `023_password_reset_requests`
- SQL: `UpdateScheduledCreditPaymentAmount`, `UpdateFutureTransactionAmount`, `UpdateSystemCategoryIcon`
- `StatsSummaryAccount` — сводка по счёту с учётом переводов; `ActivateDueFutureTransactions` при списке операций и статистике
- В ответах `Transaction`: `credit_payment_linked`; `GuardTransactionUpdate` для операций кредита
- `mapMoneyInputCursor` — стабильный курсор в полях суммы
- `setup.Handler`: `syncConfiguredMarker`, маркер `.configured` сразу после commit транзакции
- `DateTimePicker`: `calendarCells` с годом/месяцем для padding-дней; плагин `allowedHostsPlugin` (`configureServer`)
- CSS: `--bg-popover`, класс `.popover-panel` для overlay-панелей
- Версия по умолчанию: `1.1.0` (`VERSION`, `Makefile`, `Dockerfile`, OpenAPI)

## [v1.0.0]

Первый стабильный релиз self-hosted приложения для личного учёта финансов.

### Добавлено

**Установка и инфраструктура**

- Три способа установки: сборка из исходников (`make build`), Docker (GHCR), бинарники GitHub Releases (Linux, Windows, macOS; amd64 и arm64)
- Первый запуск через `/setup`: администратор, пароль с подтверждением, опциональная регистрация
- Автоматические миграции SQLite (goose) при старте; схема `001`…`019` (одна таблица — один файл)
- Синхронизация версии приложения в БД (`app_version` / `previous_app_version`) при обновлении бинарника
- Переменные окружения (`BUHGALTER_ADDR`, `BUHGALTER_DATA_DIR`, `BUHGALTER_LOCALES_DIR`, …)
- Пример nginx reverse proxy + HTTPS; поле **внешний URL** в админке для работы за прокси
- CI/CD: тесты, линтеры, сборка Docker-образа при git tag; GoReleaser для релизов

**Пользователи и безопасность**

- Регистрация (если разрешена), сессии в cookie (10 дней неактивности)
- API-токены (Bearer), `GET /auth/verify`
- Правила сложности пароля при setup, смене пароля и создании пользователя
- Изоляция данных по `user_id`; админ-панель: пользователи, системные настройки, бэкапы, восстановление, диагностика
- Секретный ключ шифрования токенов уведомлений (set-only в админке)
- Audit log в `logs/audit/YYYY-MM-DD.jsonl`

**Финансовый учёт**

- Счета: банковские (справочник банков РФ) и наличные, архивация, основной счёт (★)
- Категории и подкатегории доходов/расходов с иконками, drag-order, главная категория на тип
- Операции: расход, доход, переводы; плановые операции (`future`) с активацией по дате
- Двойной баланс на главной: текущий и прогноз с учётом будущих операций за месяц
- Долги: единый список должников, погашение, виджет на главной, уведомления по срокам
- Кредиты: график платежей, автосписание в полночь по timezone пользователя, досрочные платежи, закрытие и удаление

**Импорт, экспорт, аналитика**

- Импорт CSV/XLSX с маппингом колонок (preset Cubux)
- Экспорт операций в CSV
- Статистика по типам, категориям и периодам; поиск по операциям и комментариям
- Контекстная сводка на страницах со списками операций

**Уведомления**

- Telegram и MAX (official API `platform-api2.max.ru`); настраиваемые шаблоны и время отправки
- Уведомления по долгам («я должен» / «мне должны») с учётом timezone

**Интерфейс**

- SvelteKit UI: светлая и тёмная тема, русский и английский (svelte-i18n)
- Настройки: профиль, пароль, счета, категории, импорт, уведомления, API-токены
- Адаптивная вёрстка, PWA (manifest, service worker, иконки)
- Toast-уведомления, пустые состояния, единые диалоги (без `window.confirm`)

**API и документация**

- REST API `/api/v1/…` с полным OpenAPI (описания на русском)
- Интерактивная документация: Redoc на `/docs`
- Документация в `docs/`: установка, модель данных, UI-гайды, импорт Cubux

**База данных**

- Все изменения в базе данных - через миграции

**Локализация**

- Локали сервера вынесены из бинарника (`server/locales/`, volume в Docker)

### Техническое

- Стек: Go 1.26+, SQLite, SvelteKit, встроенный статический фронтенд (`embedstatic`)
- Команда `make version vX.Y.Z` — единая простановка semver во всех артефактах (`VERSION`, OpenAPI, Dockerfile, …)
