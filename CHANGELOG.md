**UI/UX**

### Изменено

**UI/UX**

- **Периодические операции** (`/recurring-operations`): время выполнения по умолчанию — **08:00** в часовом поясе пользователя (раньше 00:00; UI и API при пустом `time_local`)
- **Поля суммы (`MoneyInput`):** пустое и нулевое значение не отображается как `0.00` — показывается placeholder `0.00`; при blur нулевой ввод очищается (`formatMoneyForInput`, `formatMoneyInput`); единообразно в формах счетов, операций, переводов, кредитов, долгов и периодических операций
- **Форма перевода:** на странице счёта в «Откуда» подставляется текущий счёт; на главной и в «Все операции» — основной
- **Форма перевода:** выбранный в одном поле счёт не показывается в противоположном селекте («Откуда» / «Куда»); при совпадении значений автоматически подбирается другой счёт (`transferAccountOptions`, `pickOtherAccountId` в `$lib/transfer-accounts.ts`)

### Исправлено

**Backend**

- **Периодические операции:** после автоматического создания операции планировщиком вызывается `accountbalance.Refresh` — пересчитывается `current_balance` счёта. Раньше транзакция появлялась в списке, но баланс не менялся (с v1.2.2 баланс денормализован и требует явного refresh)
- **Автосписание кредита:** операция сохраняется с датой/временем из `debit_time_local` в часовом поясе пользователя (`paymentTransactionDate`; раньше в БД попадало 00:00)

**UI/UX**

- **Кредиты:** в графике платежей и списке кредитов дата следующего платежа показывается с временем автосписания (`formatCreditPaymentDateForDisplay`, `debit_time_local`), а не полуночью
- **Админка:** кнопки «Поддержать проект» и «Репозиторий» снова видны во вкладке «Администрирование» в настройках (с v1.2.0 админка встроена в `/settings?tab=admin` без `admin/+layout.svelte`, где жили ссылки)

## [v1.2.2] — 2026-06-28

> **ОБЯЗАТЕЛЬНО СДЕЛАЙТЕ БЕКАП!** Перед обновлением сохраните копию базы (`data/buhgalter.db`) и каталога `backups/`.

### Добавлено

**Обновления**

- `GET /api/v1/version/check` — сравнение текущей версии с последним релизом на GitHub (кеш на сервере 24 ч)
- Веб-клиент: раз в сутки проверка обновления; модальное предупреждение о бэкапе и release notes — **только для администраторов**; ссылка «скачайте бекап» → **Настройки → Админка → Бекапы**

**Производительность и API**

- Миграция `028`: колонка `accounts.current_balance` — денормализованный баланс счёта; индекс `idx_tx_user_account_balance` на `transactions`
- `accountbalance` — пересчёт и хранение `current_balance` при изменении операций; `BackfillAll` при миграции
- Пакетные SQL-запросы для дашборда и прогноза (`ForecastsByUser`, агрегаты future-операций по всем счетам за один проход)
- In-memory кеш GET-ответов на сервере (`apicache`): TTL 5 мин для справочников, 1 мин для остальных; инвалидация при любой мутации пользователя ([docs/ui-api-cache.md](docs/ui-api-cache.md))
- `GET /api/v1/ui/meta` — агрегированные справочники для старта веб-клиента (счета, банки, категории, должники, кредиты); банки из ответа попадают в клиентский кеш на 24 ч

**Фоновые задачи**

- [Активация future-операций в фоне](roadmap/activate-future-transactions.md): `FutureRunner` в `scheduler` — раз в минуту и при старте сервера; допустимый лаг актуальности — до ~1 мин

**UI/UX**

- Главная (`/`), **Все операции** (`/transactions`), **страница счёта** (`/accounts/[id]`): три кнопки-иконки **Доход / Расход / Перевод** (`NewTransactionButtons`, Font Awesome Solid: `f067`, `f068`, `f0ec`); подписи только в `title` / `aria-label`
- На странице **Все операции** добавлена кнопка **Перевод** (раньше был только доход/расход через общую «Операция»)
- Форма новой операции (`TransactionForm`): переключатель типа убран — тип задаётся кнопкой; заголовок модалки «Доход» или «Расход»; при редактировании тип по-прежнему только текстом
- На странице счёта кнопки операций вынесены из меню «⋯» в шапку карточки (десктоп); на **мобильных** — в меню «⋯»; в меню — редактирование, основной, архив, удаление
- `/stats` → «По категориям»: отдельные секции **доходы** и **расходы**; процент считается внутри каждого типа

### Изменено

**Производительность**

- Баланс счёта читается из `current_balance`, а не пересчитывается на каждый запрос
- `ActivateDueFutureTransactions` убран из hot-path `List`, `DashboardForUser`, `stats.Summary` — активация только в фоне

**UI/UX**

**Документация и контракты**

- OpenAPI: уточнено описание `percentage` в `StatsCategoryItem`; схема `UIMetaResponse` для `GET /ui/meta`
- [docs/ui-api-cache.md](docs/ui-api-cache.md) — серверный in-memory кеш GET; клиент кеширует только статические банки

### Исправлено

**UI/UX**

> При деплое через Docker и переходе на каталоги `./data`, `./backups`, `./logs` на хосте — [инструкция по миграции](docs/install/docker.md#docker-bind-mount-migration).

Подробные release notes для пользователей: [docs/release-notes-v1.2.1.md](docs/release-notes-v1.2.1.md).

### Добавлено

**Инфраструктура и API**

- `BUHGALTER_LOG_MODE` (`prod` / `dev`): в `prod` редактируются чувствительные заголовки, в `dev` — полные request-логи и расширенная диагностика ошибок
- Docker Compose по умолчанию монтирует `./data`, `./backups`, `./logs` с хоста (настраивается через `BUHGALTER_HOST_*_DIR` в `.env`)
- Docker-образ: entrypoint при старте создаёт каталоги volume и выставляет владельца **uid 1000** (`buhgalter`) — ручной `chown` на хосте не нужен
- В JSON-ошибках API опциональное поле `error.field` — подсказка, какое поле запроса некорректно

**Кредиты и ипотека**

- График ипотеки с **ежедневным начислением процентов**: отдельный расчёт аннуитетного платежа и генерация (N−1 равных платежей + закрывающий)
- Ручной ввод ежемесячного платежа ипотеки из договора банка (`monthly_payment` в API и поле в форме создания)
- `POST /api/v1/credits/schedule/preview` — предпросмотр графика с `calculated_monthly_payment` без сохранения

**UI/UX**

- Периодические операции: список на мобильных — карточки вместо горизонтальной прокрутки таблицы; форма вынесена в `{#snippet}`
- Редактирование операций дохода/расхода: меню «⋯» → **Изменить** на главной, `/transactions`, странице счёта; форма `TransactionForm` — счёт, категория, подкатегория, сумма, описание, дата; тип (доход/расход) не меняется; дата в будущем → `kind=future`
- Оплата кредита: в модалке можно выбрать **счёт списания** только для данного платежа (`account_id` в API); основной счёт кредита (`debit_account_id`) не меняется
- Действия в строках — компонент `RowActionsMenu`: меню «⋯» с иконкой и подписью у каждого пункта (операции, периодические, категории, долги, график платежей)
- Список счетов `/accounts`: меню «⋯» на каждой карточке — редактировать (inline на карточке с подписями полей), сделать основным, архивировать, удалить
- Страница счёта: действия (новая операция, перевод, редактирование, основной, архив, удаление) — в меню «⋯» в шапке карточки
- Настройки → Категории: «Сделать главной» — в меню «⋯»; раскрытие подкатегорий — клик по названию с ▶/▼ после текста
- Страница кредита: «Оплатить» — отдельная кнопка слева от меню «⋯» в шапке; в строках графика оплата остаётся в меню
- Страница должника: вкладки «Активные / Закрытые» через `PageTabs` (как в настройках и списке долгов)
- Фильтры (`FilterPanel`): на мобильных свёрнуты под спойлер «Фильтры» с chevron; на десктопе всегда развёрнуты (`/transactions`, `/stats`, страница счёта)
- In-memory кеш справочников в веб-клиенте: `GET /banks`, `GET /categories`, `GET /debtors` — TTL 5 мин, дедупликация параллельных запросов, инвалидация при изменениях и logout

### Изменено

**UI/UX**

- `/stats`: в блоке «По категориям» только таблица категорий (отдельный список по подкатегориям убран)
- `/stats`: блок «По периодам» — человекочитаемые подписи (месяц «Июнь 2026», неделя «1–7 июня 2026», день — `YYYY-MM-DD`); порядок от новых к старым
- Настройки → Категории: иконка категории/подкатегории — декоративная (без клика); главная категория отмечена галочкой у названия

- Погашение долга (`SettleDebtForm`): переключатель «Не учитывать в балансе» (по умолчанию выключен) — затем счёт списания; при включённом переключателе поле счёта скрыто; счёт по умолчанию — из долга (`account_id`) или основной счёт; `affects_balance` при погашении задаётся в форме, а не наследуется от создания долга
- Модалка оплаты кредита: убрана кнопка «Сегодня» у поля даты (дата задаётся календарём или берётся из графика)

**Интерфейс**

- SvelteKit UI: светлая и тёмная тема, русский и английский (svelte-i18n)
- Настройки: профиль, пароль, счета, категории, импорт, уведомления, API-токены
- Адаптивная вёрстка, PWA (manifest, service worker, иконки)
- Toast-уведомления, пустые состояния, единые диалоги (без `window.confirm`)

**API и документация**

- REST API `/api/v1/…` с полным OpenAPI (описания на русском)
- Интерактивная документация: Redoc на `/docs`
- Документация в `docs/`: установка, модель данных, UI-гайды, импорт Cubux

**База данных**

- Все изменения в базе данных - через миграции

**Локализация**

- Локали сервера встроены в бинарник по умолчанию; файловые `ru.json`/`en.json` используются только как опциональное переопределение

### Техническое

- Стек: Go 1.26+, SQLite, SvelteKit, встроенный статический фронтенд (`embedstatic`)
- Команда `make version vX.Y.Z` — единая простановка semver во всех артефактах (`VERSION`, OpenAPI, Dockerfile, …)
