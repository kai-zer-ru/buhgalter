## Модерация пользователей

При **самостоятельной регистрации** новый пользователь получает статус **«На проверке»** — войти он не сможет, пока администратор не активирует учётную запись.

| Статус | Поведение |
|--------|-----------|
| На проверке | ожидает решения админа |
| Активен | полный доступ |
| Заблокирован | вход и API запрещены, сессии сбрасываются |

В **админке → Пользователи**:

- колонка «Статус»;
- меню «⋯»: активировать, заблокировать, разблокировать;
- плашка со списком ожидающих модерации (как при запросе сброса пароля);
- переход по ссылке `/admin/users?moderate={id}` из уведомления.

Администраторам с настроенным Telegram/MAX приходит уведомление о новой регистрации (триггер **«Регистрация пользователя»**). Нужен **внешний URL** в админке — иначе ссылка на модерацию в сообщении не формируется.

Подробнее — [api/user-status.md](api/user-status.md).

## Кредитная карта

Новый тип счёта **«Кредитная карта»** — те же операции, что у банковского счёта, плюс:

- обязательный **лимит**;
- блок **«Кредитный баланс»** на главной (карты не входят в общий баланс);
- кнопки **«Оплатить»** и **«Списать комиссию»**;
- предупреждение при расходе, если баланс уйдёт в минус.

Не путать с разделом **«Кредиты»** (займы с графиком платежей).

Подробнее — [ui-credit-cards.md](ui-credit-cards.md).

## Учёт суммы кредита в балансе

При **создании потребительского кредита** (не ипотека, не рассрочка) можно включить переключатель **«Учитывать доход в балансе»** — на счёт списания сразу попадёт операция дохода на сумму займа.

Переключатель **недоступен**, если в графике есть платёж с датой в прошлом. После сохранения кредита опцию изменить нельзя.

## Бюджет

Новый раздел **«Бюджет»** в верхнем меню (`/budget`):

- лимиты на категорию, подкатегорию или все расходы **за календарный месяц**;
- прогресс «план vs факт», копирование лимитов между месяцами;
- уведомления при достижении порога (по умолчанию 90%) и 100%;
- виджет на главной; колонки «Бюджет» / «Остаток» в статистике по категориям.

Подробнее — [budget.md](budget.md), [ui-budget.md](ui-budget.md).

## Архивация и удаление счётов

- **Архив** — счёт скрыт из активных, новые операции недоступны, история сохраняется.
- **Удаление** — мягкое (`status = deleted`), вкладка **«Удалённые»** на `/accounts`; просмотр истории по старой ссылке.

Для **наличных и банковских** счетов с остатком при архивации/удалении предлагается **перевести деньги** на другой активный счёт.

Для **кредитной карты** перевод не нужен — закрыть можно только при полном погашении (`баланс ≥ лимит`).

Подробнее — [accounts-archive-delete.md](accounts-archive-delete.md).

## Автопополнение банковского счёта

Для счёта типа **«Банк»** можно настроить автопополнение: при падении баланса ниже порога — перевод с другого банковского счёта до целевой суммы (категория «Перевод», комментарий «Автопополнение»).

На главной у банковских счетов — индикатор **«Автопополнение: ✓/✗»**. При нехватке средств на счёте-источнике автопополнение отключается, приходит уведомление.

Подробнее — [balance-maintenance.md](balance-maintenance.md), [ui-balance-maintenance.md](ui-balance-maintenance.md).

## Уведомления в Telegram / MAX

### Нехватка средств

При включённой настройке **«Отрицательный баланс»** к напоминаниям о кредитах, плановых расходах и долгах «я должен» дописывается суффикс вида «На балансе не хватает 1 200 ₽!». Шаблон суффикса редактируется отдельно.

### Ссылки в сообщениях

Во всех шаблонах появились плейсхолдеры со ссылками на нужный раздел (кредит, долги, бюджет, операции, модерация и т.д.). Базовый адрес — **внешний URL** в админке; если он не задан — в тексте подсказка настроить его.

Полная таблица плейсхолдеров — [notifications.md](notifications.md#ссылки-в-шаблонах).

## Форматирование сумм

Все суммы в интерфейсе выводятся единообразно: разделитель тысяч — пробел, дробная часть — точка, символ валюты (например `12 000.00 ₽`). Для разработчиков — компонент `MoneyDisplay` и [transactions-display.md](transactions-display.md#формат-денег).

## UI

### Верхнее меню на мобильных

Гамбургер-меню открывает компактную панель под шапкой. Пункты **«Настройки»** и **«Админка»** ведут на **отдельный экран подменю** с кнопкой «Назад» — длинный список ссылок больше не раскрывается в одном столбце.

### Главная

На **мобильных** список счетов свёрнут в спойлер **«Счета (N)»**; на десктопе — сетка карточек как раньше.

### Статистика

Блоки «По категориям» и «По периодам» — **на всю ширину** друг под другом (категории выше периодов). Сводка — три карточки в ряд на широком экране.

### Ширина страницы

На десктопе основной контент шире — до **80rem** (`max-w-5xl`).

## Исправления

- **Долги:** при удалении единственной операции «Взял в долг» / «Дал в долг» долг удаляется вместе с ней; при удалении погашения остаток пересчитывается; **нельзя** удалить начальную операцию, если по долгу есть погашения.
