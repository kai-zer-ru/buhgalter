## Android-приложение

Первый полноценный **мобильный клиент** (APK в [GitHub Releases](https://github.com/kai-zer-ru/buhgalter/releases)):

- **Установка:** скачайте `buhgalter-android-1.4.0.apk` (universal) или сборку под архитектуру (`-arm64-v8a`, `-armeabi-v7a`, `-x86_64`). Android **8.0+**.
- **Вход:** **логин и пароль** или **API-токен** из веб-интерфейса (Настройки → API-токены) — на выбор после адреса сервера.
- **Первый запуск:** поиск сервера в Wi‑Fi (mDNS + сканирование подсети) или ручной ввод **LAN URL**; проверка `GET /api/v1/health`.
- **Экраны:** главная, счета, операции, переводы, долги, кредиты (**включая создание** по шагам), бюджет, статистика, настройки, админка (без управления пользователями).
- **Офлайн:** очередь изменений (операции, переводы, категории, долги, счета, бюджет) с отправкой при появлении сети; кеш справочников на устройстве.
- **Блокировка:** PIN и биометрия (Настройки → Безопасность). При **выходе**, отключении сервера или истечении сессии блокировка **сбрасывается**.
- **Виджеты** на рабочем столе: быстрые действия, баланс, бюджет, ближайшие платежи, один счёт.
- **«Поделиться»** из других приложений (текст или картинка) открывает форму нового расхода.
- **Настройки → Сервер:** внешний URL, до 5 домашних Wi‑Fi, fallback LAN→внешний, доверие к self-signed HTTPS, экспорт очереди офлайн, **отладочные логи** (сохранение в «Загрузки»).
- Ярлыки лаунчера: расход / доход / перевод.
- Status bar и navigation bar подстраиваются под светлую/тёмную тему приложения; иконка поддерживает тематические (Material You) иконки ОС.
- **Сохранить** в настройках и админке — остаётесь на том же экране. После **добавления** операции, перевода, долга, счёта или кредита — возврат назад без повторного открытия формы кнопкой «Назад».

Управление пользователями — по-прежнему только в **веб-интерфейсе**.

Подробнее — [android-client.md](android-client.md), [android/README.md](../android/README.md).

## Сервер — discovery, health, локализация

- При старте сервер публикует себя в LAN через **mDNS** (`_buhgalter._tcp`). Отключение: `BUHGALTER_MDNS_ENABLED=false` в `.env`.
- `GET /api/v1/health` может возвращать **`external_url`** — внешний URL из админки. Android показывает домен в списке найденных серверов.
- `GET /api/v1/ui/i18n/{lang}` — каталог строк UI (для Android, если версия APK отстаёт от сервера).
- `GET /api/v1/budgets/spent-preview` — уже потраченная сумма по выбранному scope (для формы бюджета).
- Новые переменные: `BUHGALTER_MDNS_ENABLED` (по умолчанию `true`), `BUHGALTER_MDNS_NAME`.

**Миграция** `042_primary_not_credit_card.sql`: снимает ошибочный «основной» флаг с кредитных карт и при необходимости назначает основной среди наличных/банковских счетов. Тема по умолчанию `system` задаётся в коде при создании пользователя и в `schema.sql` для новых БД; существующие профили со светлой/тёмной темой не меняются.

## Тема оформления

В профиле (web и Android): **Как на устройстве** / светлая / тёмная. «Как на устройстве» — значение по умолчанию; интерфейс следует системной светлой или тёмной схеме (`prefers-color-scheme`).

## Веб-интерфейс

- **Stale-while-revalidate:** при открытии страницы данные сразу из локального кеша браузера, обновление с сервера — в фоне.
- Прогрев кеша после входа; точечное обновление экранов при фоновом refresh.
- При сбое **первой** загрузки экрана — сообщение и «Повторить», а не пустой экран.
- **Новый счёт:** после сохранения — возврат на список (или экран-источник), не на карточку нового счёта.
- **Статистика → По категориям:** подкатегории под спойлером у каждой категории (то же в Android).
- Пункт **«Главная»** снова в верхнем меню (первым).
- **Периодические операции:** кнопка «Добавить» в шапке (рядом с заголовком), форма создания под шапкой — как на других списках (бюджет, счета, кредиты).

Подробнее — [ui-api-cache.md](ui-api-cache.md), [ui-stats.md](ui-stats.md), [ui-navigation.md](ui-navigation.md).

## Счета

**Начальный баланс при редактировании (веб):** поле берётся из `initial_balance`, а не из текущего баланса на экране (раньше можно было случайно перезаписать стартовый остаток). В Android-клиенте форма сразу использует `initial_balance`.

**«Мои средства»** на главной и в списке счетов: сначала наличные, затем банки; внутри типа основной счёт — первым.

**Кредитную карту нельзя сделать основным счётом** (меню «⋯» и API). Если раньше карта была основной — миграция `042` это исправит.

Подробнее — [data-model.md](data-model.md), [ui-credit-cards.md](ui-credit-cards.md).

## Бюджет

В форме добавления/редактирования лимита (веб и Android) под полем суммы показывается **«Уже потрачено»** — факт по выбранной области (все расходы / категория / подкатегория и счёт), если открыт **текущий или прошлый** месяц. На будущий месяц подсказка скрыта.

Подробнее — [ui-budget.md](ui-budget.md), [budget.md](budget.md).

## Кредиты

- Переключатель **«Уже платил по графику»** виден только если в предпросмотре есть платежи с датой в прошлом (веб и Android).

Подробнее — [ui-credits.md](ui-credits.md).

## UI — выпадающие списки

Исправлено позиционирование списков **Select** / **Combobox** и календаря без portal: панель больше не «висит» над полем с зазором (часто на Android в **Настройки → Профиль** — тема, часовой пояс). Обёртка `.relative` только вокруг контрола; подпись и подсказка — снаружи.

Подробнее — [ui-dialogs.md](ui-dialogs.md).

## Уведомления — формулировки по долгам

Исправлена формулировка напоминания `debt_due_soon`:

- Долг **«мне должны»** — «получить долг от …», а не «вернуть долг …».
- В день срока — **«сегодня»**, а не «через 0 дн.».

В шаблоне появились плейсхолдеры `{action}` и `{when}`. Если ранее сохраняли кастомный текст — сбросьте к дефолту в настройках уведомлений или поправьте вручную.

Подробнее — [notifications.md](notifications.md).

## Обновление

**Сервер / Docker:** бэкап → замена бинарника или `docker compose pull && up -d` → миграция `042` применится автоматически при старте.

**Android:** установите новый APK поверх старого или скачайте из попапа «Версии» в боковом меню (если версия приложения отстаёт от сервера по мажор/минор). При небольшом отставании патча UI-строки могут подтянуться с сервера без обязательной переустановки.
