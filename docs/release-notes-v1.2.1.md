# Release notes — v1.2.1

Краткая сводка для пользователей и администраторов. Полный список изменений — [CHANGELOG.md](../CHANGELOG.md).

---

## Инфраструктура

- **Docker:** по умолчанию база, бэкапы и логи лежат на хосте в `./data`, `./backups`, `./logs` (каталог `docker/`). Пути настраиваются через `BUHGALTER_HOST_DATA_DIR`, `BUHGALTER_HOST_BACKUPS_DIR`, `BUHGALTER_HOST_LOGS_DIR`. Инструкция по переносу с named volumes — [install/docker.md](install/docker.md)
- **`BUHGALTER_LOG_MODE`:** `prod` — безопасные логи (редактирование чувствительных заголовков); `dev` — полные request-логи и расширенная диагностика при ошибках

## API и ошибки

- Ответы с ошибками могут содержать поле **`error.field`** — имя поля запроса, к которому относится сообщение (например `monthly_payment`, `term_months`)
- Документированы `POST /credits/schedule/preview`, схема `PreviewScheduleResponse` и коды `ERR_CREDIT_*` в OpenAPI
- В OpenAPI уточнено поведение `GET /dashboard` (`forecast_balance`, `has_future_this_month`) и lifecycle оплаты/удаления `POST/DELETE /credits/{id}/payments*`
- В OpenAPI добавлены endpoints `/stats/*` (сводка, категории, периоды, поиск, контекст); у `GET /stats/by-period` — формат ключей `period` и сортировка от новых к старым
- OpenAPI: поведение in-memory кеша веб-клиента для `GET /banks`, `GET /categories`, `GET /debtors` — [ui-api-cache.md](ui-api-cache.md)
- OpenAPI: `POST /debts/{id}/settle` — схема `SettleDebtRequest`; `affects_balance` и `account_id` задаются при погашении независимо от флага долга при создании
- OpenAPI: `PUT /transactions/{id}` — тип операции не меняется (`ERR_TX_TYPE_CHANGE`); дата в будущем → `kind=future`
- OpenAPI: `POST /credits/{id}/payments` — опциональный `account_id` (счёт только для данной оплаты)
- OpenAPI: привязка UI-меню счетов на `/accounts` к endpoints `PUT/DELETE /accounts/{id}`, `POST .../archive`, `POST .../primary`

## Интерфейс

### Действия и меню

- **Строки таблиц и карточек:** вместо ряда иконок — меню «⋯» с подписанными пунктами (иконка + текст). Применено к операциям, периодическим, категориям, долгам, графику платежей
- **Главная и «Все операции»:** в «Последних операциях» и на `/transactions` — меню «⋯» (сделать периодической, изменить, удалить); переводы редактируются через форму перевода
- **Редактирование операции:** счёт, категория, подкатегория, сумма, описание, дата; тип доход/расход не меняется; дата в будущем — плановая операция (`future`)
- **Список счетов `/accounts`:** на каждой карточке — меню «⋯» (**Редактировать**, **Сделать основным**, **Архивировать**, **Удалить**); редактирование — inline на карточке (поля с подписями: название, банк, начальный баланс)
- **Страница счёта:** все действия — в меню «⋯» в шапке; форма редактирования с теми же подписями полей
- **Настройки → Категории:** «Сделать главной» — в меню «⋯»; подкатегории раскрываются кликом по названию (▶/▼ справа от текста); иконка категории декоративная
- **Страница кредита:** кнопка **«Оплатить»** вынесена отдельно слева от меню «⋯» в шапке; в строках графика «Оплатить» остаётся в меню строки

### Мобильная вёрстка

- **Периодические операции:** список — карточки, форма создания/редактирования адаптирована под узкий экран
- **Фильтры** (операции, статистика, страница счёта): на мобильных свёрнуты под спойлер «Фильтры» с иконкой раскрытия; на десктопе панель всегда видна
- **Вкладки должника** («Активные / Закрытые»): тот же стиль подчёркивания, что в настройках и списке долгов

### Статистика

- В блоке **«По категориям»** остаётся только таблица категорий; отдельный список по подкатегориям убран
- **«По периодам»:** подписи в читаемом виде — по месяцам «Июнь 2026», по неделям «1–7 июня 2026», по дням — дата `YYYY-MM-DD`; список от новых к старым

### Прочее UI

- **Кеш справочников:** банки, категории и должники кешируются в памяти браузера (5 мин); при редактировании категорий, создании должника или выходе кеш сбрасывается. Счета, операции и дашборд всегда запрашиваются заново
- **Погашение долга:** переключатель «Не учитывать в балансе» (по умолчанию выключен), затем счёт списания; при включённом переключателе счёт скрывается; по умолчанию — счёт начальной операции долга или основной счёт; работает для любого долга, в том числе созданного «без баланса»
- **Пагинация графиков кредита:** кнопки «В начало» и «В конец» (форма создания и страница кредита)
- **Календарь:** `DateTimePicker` стабилизирован — без скачков высоты между месяцами и состояниями поля

## Кредиты и ипотека

- **Ипотека — расчёт графика:** ежедневное начисление процентов; автоматический аннуитет может отличаться от суммы в договоре банка
- **Платёж из договора:** в форме создания можно вручную указать ежемесячный платёж банка — график пересчитается (N−1 одинаковых + закрывающий)
- **Потребительские кредиты:** ручной платёж проверяется (слишком низкий / слишком высокий для срока); для ипотеки принимается любая положительная сумма
- **Предпросмотр графика:** при ошибке валидации предыдущие строки не сбрасываются
- **График на странице кредита:** оплатить — только ближайший неоплаченный; удалить — только последний оплаченный; удалённый платёж восстанавливается как неоплаченный
- **Оплата кредита:** в модалке можно выбрать счёт списания только для этого платежа; основной счёт кредита не меняется; кнопка «Сегодня» у даты убрана

## Исправления

- Создание ипотеки и preview графика больше не дают **500** при некорректных параметрах — возвращается **400** с понятным текстом
- Исправлен отрицательный последний платёж в длинном графике ипотеки
- Исправлен пустой график при корректном расчёте (гонка запросов preview)
- Ручной платёж, совпадающий с расчётным в пределах 1 ₽, не считается override
- Исправлен регресс: платёж по кредиту с будущей датой снова корректно влияет на прогноз баланса текущего месяца (главная и страница счёта)
- Устранён преждевременный перевод `future`-операций кредита в `manual` до наступления даты операции
- Исправлена иконка пункта «Изменить банк» в меню кредита (галочка → иконка банка)
- Исправлено исчезновение фильтров на десктопе после введения мобильного спойлера
