# UI — кредитные карты

Тип счёта **`credit_card`**: учёт кредитной карты как обычного счёта с лимитом и отдельным блоком на главной.

**Не путать с модулем «Кредиты»** ([ui-credits.md](ui-credits.md)): целевые займы с графиком (ипотека, потребкредит, рассрочка). Кредитная карта — отдельный тип **счёта**, к кредитам не относится.

Связанные документы: [data-model.md](data-model.md), [import/cubux.md](import/cubux.md), [api/openapi.yaml](api/openapi.yaml).

---

## Счёт и баланс

| | Наличные / банк | Кредитная карта |
|---|---|---|
| Тип API | `cash`, `bank` | `credit_card` |
| Баланс | деньги на счёте | баланс карты — то же поле `balance` |
| Лимит | — | обязательное поле `credit_limit` |
| Расход | уменьшает баланс | уменьшает баланс |
| Доход | увеличивает баланс | увеличивает баланс |
| Погашение | — | перевод **на** карту с другого счёта |

Типичный сценарий: зарплата на дебетовый счёт, покупки с кредитной карты, раз в месяц ручной перевод с дебета на карту.

**Баланс карты** хранится в `accounts.current_balance` (копейки), как у `cash` и `bank`. В UI отображается без знака «−». При создании **начальный баланс по умолчанию 0**.

**Валюта** — валюта пользователя (`users.currency`); отдельного поля у карты нет.

**Название** — обычное поле `name` (1–64 символа), задаётся пользователем («Тинькофф Black», «Кредитка жены»).

Автоматических списаний, графика погашения, расчётного и льготного периода, минимального платежа **нет**.

## Модель данных

Поля на таблице `accounts` (см. [data-model.md](data-model.md)):

| Поле | Описание |
|------|----------|
| `type` | `credit_card` |
| `bank_id` | банк-эмитент (обязателен; иконка в UI, как у `bank`) |
| `credit_limit` | лимит, копейки (обязателен при создании) |
| `payment_account_id` | счёт по умолчанию для погашений (`cash` / `bank`, опционально) |

Операции — те же типы, что у любого счёта:

- **Расход** — `expense` на счёт карты
- **Доход** (возврат) — `income` на счёт карты
- **Погашение** — `transfer` с дебетового счёта на карту

## Создание и редактирование

Страницы: `/accounts/new`, `/accounts`, `/accounts/{id}`.

Тип **«Кредитная карта»** — третья вкладка в форме (рядом с «Наличные» и «Банковский»):

| Поле | Поведение |
|------|-----------|
| Банк | выбор из справочника `GET /api/v1/banks` |
| Лимит | обязательно (`MoneyInput`) |
| Начальный баланс | по умолчанию `0`; кнопка **«Лимит»** подставляет значение лимита |
| Счёт для переводов | опционально: `cash` / `bank` для погашений по умолчанию |

Если **счёт для переводов** не задан, при «Оплатить» подставляется основной счёт (`is_primary`), иначе первый активный `cash` / `bank`.

Иконка счёта (`AccountIcon`): логотип банка + бейдж «КК», чтобы отличать от дебетового счёта того же банка.

**Архивация** — как у остальных счетов (`POST /accounts/{id}/archive`). Кредитную карту можно архивировать только при **полном погашении** (`balance >= credit_limit`); иначе API вернёт `ERR_CREDIT_CARD_ARCHIVE_NOT_FULLY_PAID`.

## Карточка и список счетов

На `/accounts/{id}` и в карточке на `/accounts`:

- **баланс** и **лимит** — отдельные строки;
- в меню «⋯» (только для `credit_card`): **«Оплатить»**, **«Списать комиссию»**.

На главной в блоке счетов у кредитки рядом с балансом показывается лимит.

---

## Главная (`/`)

Агрегаты в `GET /api/v1/dashboard`:

| Поле | Содержимое |
|------|------------|
| `total_balance`, `total_forecast` | только `cash` + `bank` (кредитки **исключены**) |
| `credit_cards_summary` | суммарный баланс, лимит, forecast и `count` по всем активным картам |
| `accounts` | все типы, включая кредитки |

Блок **«Кредитный баланс»** — в одной строке с «Итого»; показывает суммарный баланс и лимит карт. Если карт нет (`count === 0`) — блок скрыт.

Двойной баланс (текущий / с учётом запланированных операций в текущем месяце) для карт — по тем же правилам, что для остальных счетов.

## «Оплатить»

Кнопка на странице карты и в меню на `/accounts`. Открывает форму перевода с предзаполнением:

| Поле | Значение |
|------|----------|
| Откуда | `payment_account_id` карты → иначе основной счёт → иначе первый `cash` / `bank` (можно изменить) |
| Куда | карта (поле скрыто) |
| Сумма | ввод; в UI предупреждение, если больше `лимит − баланс` |
| Комиссия | опционально |
| Дата / время | сегодня и текущее время; время под спойлером |
| Комментарий | опционально |

**Сервер** отклоняет перевод на карту, если `amount > credit_limit − balance` (`400`, код `ERR_CREDIT_CARD_PAYMENT_EXCEEDS_LIMIT`).

---

## «Списать комиссию»

Рядом с «Оплатить». Создаёт **расход** на карту:

- категория — системная **«Комиссия»**;
- комментарий — **`Комиссия за использование карты`** (подставляется автоматически).

Поля: сумма, дата, время (под спойлером).

---

## Переводы и комиссия

Погашение — обычный `POST /api/v1/transfers` на счёт карты.

При указании **комиссии** к переводу с участием кредитки автоматически подставляется комментарий к расходу в категории «Комиссия»:

| Направление | Комментарий |
|-------------|-------------|
| перевод **на** кредитку | `Комиссия за использование карты` |
| перевод **с** кредитки | `Комиссия за перевод` |

---

## Предупреждения в UI

| Ситуация | Поведение |
|----------|-----------|
| Расход на карту, после которого баланс станет отрицательным | мягкое предупреждение в форме операции; **submit не блокируется** |
| «Оплатить» сверх `лимит − баланс` | предупреждение в форме; **сервер возвращает 400** |

---

## Статистика

Расходы и доходы по кредитке попадают в `/stats` **как по любому счёту** (по дате операции). Отдельных срезов и фильтров для кредиток нет.

---

## Импорт Cubux

На шаге «Сопоставление счетов» (`/import`) для нового счёта можно выбрать тип **«Кредитная карта»**: указать банк и лимит. Операции из файла на существующую карту маппятся так же, как на `cash` / `bank`.

Подробнее: [import/cubux.md](import/cubux.md).

## API

| Метод | Назначение |
|-------|------------|
| `POST /api/v1/accounts` | создание (`type: credit_card`, `bank_id`, `credit_limit`, опционально `payment_account_id`) |
| `PUT /api/v1/accounts/{id}` | изменение имени, банка, лимита, счёта для переводов, начального баланса |
| `GET /api/v1/dashboard` | `credit_cards_summary`, `accounts[].credit_limit` |

Схемы: `Account`, `AccountCreate`, `AccountUpdate`, `CreditCardsSummary` в [api/openapi.yaml](api/openapi.yaml).
