# UI — стабильная шапка списков

На экранах с вкладками или фильтрами элементы шапки не должны исчезать при переключении — иначе «прыгает» вёрстка.

## Правила

1. **Кнопки создания** («Новый счёт», «Дать в долг», «+ Операция» и т.п.) — **всегда в шапке** (рядом с `h1`), не зависят от активной вкладки или фильтра.
2. **Сводки и табы** — остаются на месте при смене вкладки (если данные общие — грузить отдельно от списка, см. `/debts` + `GET /debts/summary`).
3. **Повторная загрузка списка** — `filterLoading` / `tabChange`, не полный `loading` страницы:
   - список не схлопывается в одну строку «Загрузка…»;
   - при непустом списке — `opacity-60` поверх таблицы/сетки + подпись сверху;
   - при пустом списке — `EmptyStateCard` с `common.loading` и `ariaBusy`.
4. **Скрывать действие через `invisible`** — только если кнопка бессмысленна на вкладке (не для основных CTA в шапке).
5. **Одна кнопка создания** — не дублировать «Новый счёт» / «Создать» внутри `EmptyStateCard`; действие только в шапке рядом с `h1` (эталон: `/accounts`).

## Эталон

`/accounts` — одна кнопка «Новый счёт» в шапке (на любой вкладке); пустой список — только текст в `EmptyStateCard`; при смене вкладки с пустым списком — `EmptyStateCard` с `common.loading`, без мелькания CTA.

## Где применено

| Маршрут | Кнопки в шапке | `filterLoading` при вкладках |
|---------|----------------|------------------------------|
| `/accounts` | «Новый счёт» | да |
| `/debts` | «Дать в долг» / «Взять в долг» | да |
| `/debtors/[id]` | «Дать…» / «Взять…» (над табами) | нет (клиентский фильтр) |
| `/settings/categories` | форма создания под табами | нет (мгновенная смена вкладки) |
| `/transactions` | «+ Операция» | да (перезагрузка по фильтрам/пагинации) |
| `/accounts/[id]` | «+ Операция», «Перевод» | да (фильтр операций) |
| `/credits` | «+ Новый кредит» | да; вкладки и badge — [ui-credits.md](ui-credits.md) |

## Фильтры (`FilterPanel`)

Единый компонент на всех экранах с панелью фильтров (подробнее — [ui-row-actions.md](ui-row-actions.md)):

| Маршрут | Содержимое панели |
|---------|-------------------|
| `/transactions` | `TransactionFilters` |
| `/accounts/[id]` | `TransactionFilters` |
| `/stats` | собственная сетка полей |

- На **мобильных** — свёрнуты под спойлер «Фильтры» с chevron; по умолчанию закрыты.
- На **десктопе** (`≥ md`) — панель всегда развёрнута, заголовок спойлера скрыт.
- Компонент: `$lib/components/FilterPanel.svelte`; поля операций — `TransactionFilters`.

## Мобильная вёрстка

- Touch-targets минимум 44px (`layout.css`, `@media (pointer: coarse)`)
- Широкие таблицы → карточки на `md` breakpoint (кредиты, операции)
- Модалки — bottom sheet на узком экране (`ModalShell`, `ConfirmDialog`)
- Вкладки — горизонтальный скролл (`PageTabs`, `.page-tabs-scroll`)
