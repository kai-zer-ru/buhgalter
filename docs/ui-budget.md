# UI — бюджет

Страница `/budget`: лимиты расходов за календарный месяц, прогресс «план vs факт», копирование между месяцами.

Логика расчёта, API и модель данных — [budget.md](budget.md).

Связанные документы: [ui-navigation.md](ui-navigation.md), [ui-empty-states.md](ui-empty-states.md), [ui-stats.md](ui-stats.md).

---

## Навигация

Пункт **«Бюджет»** в верхнем меню (`nav.budget`), между «Периодические» и «Статистика».

Хлебные крошки: Главная → Бюджет.

Query: `?month=YYYY-MM` — выбранный месяц; по умолчанию текущий в timezone пользователя.

---

## Шапка страницы

| Элемент | Поведение |
|---------|-----------|
| **Добавить** | Открывает форму создания под шапкой; повторное нажатие — «Отмена» |
| **Скопировать** | Видна при `can_copy_from_previous`; копирует все бюджеты прошлого месяца |
| Переключатель месяца | ← / название месяца / → |

Кнопка «Добавить» только в шапке (не дублируется в пустом состоянии).

## Форма создания и редактирования

| Расположение | Когда |
|--------------|-------|
| Карточка под шапкой | Создание (`formOpen`, не `editId`) |
| Inline в карточке бюджета | Редактирование (меню «⋯» → Изменить) |

### Поля

| Поле | i18n / примечание |
|------|-------------------|
| Название | `budget.field.name` |
| Область | `category`, `subcategory`, `all_expense` |
| Лимит | `MoneyInput` |
| Категория / подкатегория | Только `expense`, без системных |
| Счёт | «Все счета» или конкретный — сужает факт, не дублирует scope |
| Порог уведомления | По умолчанию **90** %; `0` — без порога |
| Копировать на следующий месяц | `copy_forward`; подсказка `budget.field.copy_forward_hint` |
| Активен | `ToggleSwitch` |

### Вёрстка формы

Форма — явные строки (`space-y-3`), сетки `grid gap-3 md:grid-cols-N`. Область дублируется в разметке: на десктопе — в строке 1, на мобилке — отдельная строка 2 (`hidden md:block` / `md:hidden`).

#### Десктоп (`md+`)

| Строка | Содержимое |
|--------|------------|
| 1 | Название, область |
| 2 | `all_expense` — только лимит; `category` — категория + лимит; `subcategory` — категория + подкатегория + лимит |
| 3 | Счёт, порог уведомления, переключатель «Активен» |
| 4 | Переключатель копирования |

#### Мобилка

| Строка | `all_expense` | `category` | `subcategory` |
|--------|---------------|------------|---------------|
| 1 | Название | Название | Название |
| 2 | Область | Область | Область |
| 3 | Лимит | Категория | Категория |
| 4 | — | Лимит | Подкатегория |
| 5 | — | — | Лимит |
| 6 | Счёт | Счёт | Счёт |
| 7 | Порог уведомления | Порог уведомления | Порог уведомления |
| 8 | Активен | Активен | Активен |
| 9 | Копирование | Копирование | Копирование |

### Защита от дублей в форме

- «Все расходы» скрыто в списке областей, если общий бюджет уже есть в месяце.
- Из списков категорий и подкатегорий убраны уже занятые (кроме редактируемого бюджета).
- При конфликте на сервере — toast с `errors.ERR_BUDGET_DUPLICATE`.

## Карточки бюджетов

Источник: `GET /api/v1/budgets/summary?month=…`.

| Элемент | Описание |
|---------|----------|
| Порядок | `all_expense` первая, остальные по имени |
| Прогресс | `budget.progress`: потрачено / лимит |
| Прогресс-бар | Цвет по `status` (см. ниже) |
| Остаток | `budget.remaining` + процент |
| Копируется | `budget.copy_status`: да/нет (`copy_forward`) |
| По категориям | Только у `all_expense`: `budget.children` — факт/лимиты дочерних бюджетов |

### Меню «⋯»

- Изменить
- Копировать на следующий месяц (`budget.action.copy_next`)
- Удалить

---

## Цвета прогресс-бара

| `status` | Цвет | Условие |
|--------|------|---------|
| `ok` | зелёный | &lt; `alert_at_percent` |
| `warning` | жёлтый | ≥ порога и ≤ 100% |
| `exceeded` | красный | &gt; 100% |

---

## Пустое состояние

`EmptyStateCard` (`budget.empty`): «Задайте лимит на категорию, подкатегорию или все расходы за этот месяц».

Показывается, если в выбранном месяце нет бюджетов и форма создания закрыта.

## Виджет на главной

`web/src/routes/+page.svelte` — блок **«Бюджет месяца»** (`budget.widget.*`).

### Расположение

| Условие | Сетка сводки |
|---------|----------------|
| Есть кредитные карты | Строка 1: «Общий баланс» \| «Кредитный баланс»; строка 2: «Долги» (`self-start`, половина ширины на десктопе) |
| Нет кредиток | Строка 1: «Общий баланс» \| «Долги» |

Виджет бюджета — **отдельной строкой ниже** сводки (перед списком счетов), на всю ширину. Не в одной сетке с «Долгами», чтобы длинный список категорий не растягивал карточку долгов.

### Данные

`GET /api/v1/budgets/summary` без `month` — **текущий** календарный месяц в timezone пользователя.

Виджет **скрыт**, если в этом месяце нет активных бюджетов.

### Содержимое

| Зона | Что показывается |
|------|------------------|
| Шапка | Заголовок + ссылка «Подробнее →» на `/budget` |
| Основная | Только `all_expense` (если есть): имя, `spent / planned`, прогресс-бар, `%` и остаток (`budget.remaining`) |
| Спойлер | Все бюджеты `category` / `subcategory` месяца, сортировка по `%` убыв.; подпись `budget.widget.categories` — «По категориям (N)»; **свёрнут по умолчанию** (`<details>`) |

Если общего бюджета нет, а категорийные есть — видна шапка и спойлер с категориями.

Цвета прогресс-бара — те же, что на `/budget` (`ok` / `warning` / `exceeded`).

### Поведение

- Карточка не целиком ссылка: переход на `/budget` — только по «Подробнее»; раскрытие спойлера не ведёт на страницу бюджета.
- При большом числе категорий высота сводки («Долги») не меняется — список внутри спойлера.

## Статистика

`/stats` — в «По категориям» колонки «Бюджет» и «Остаток» при `scope=category` за период, пересекающийся с месяцем отчёта. См. [ui-stats.md](ui-stats.md).
