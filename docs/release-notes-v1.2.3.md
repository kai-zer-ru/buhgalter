## Операции — «Повторить»

На **главной**, в **«Все операции»** и на **странице счёта** в меню «⋯» каждой строки добавлен пункт **Повторить**:

- Открывается форма **новой** операции с полями из выбранной строки (счёт, категория, сумма, описание; для перевода — счета, сумма, комиссия)
- Дата подставляется **текущая** (сегодня)
- Работает для **дохода**, **расхода** и **перевода**
- Недоступно для платежей по кредиту и для дохода/расхода в **системных категориях** (как «Сделать периодической»)

Подробнее — [transactions-display.md](transactions-display.md), [ui-row-actions.md](ui-row-actions.md).

## Форма перевода

### Счёт по умолчанию в «Откуда»

- На **странице счёта** (в том числе «Наличные»): при нажатии **Перевод** в поле **Откуда** автоматически подставляется **текущий счёт**, а не основной
- На **главной** и в **«Все операции»** — по-прежнему подставляется **основной** счёт

### Исключение счёта из второго селекта

В селектах **«Откуда»** и **«Куда»** выбранный в одном поле счёт **не показывается** в другом — нельзя выставить перевод «с Яндекса на Яндекс» в интерфейсе (раньше можно было, сервер возвращал ошибку).

Если значения совпали (например, остался один активный счёт), второе поле подбирается автоматически.

Подробнее — [transactions-display.md](transactions-display.md).

## Поля суммы

Во всех формах с компонентом **MoneyInput** пустое и нулевое значение больше не показывается как `0.00` — поле остаётся пустым с подсказкой `0.00`. При потере фокуса нулевой ввод очищается. При сохранении пустое поле по-прежнему означает ноль.

Затронуты формы:

- счета (создание, редактирование на `/accounts` и странице счёта)
- доход / расход / перевод
- кредиты (сумма, взнос, платёж)
- долги и погашение
- периодические операции

Раньше поведение было неоднородным: где-то уже был placeholder, где-то отображался текст `0.00`, где-то поле оставалось пустым.

Подробнее — [transactions-display.md](transactions-display.md).

## Периодические операции

- Время выполнения по умолчанию — **08:00** в часовом поясе пользователя (раньше 00:00; при пустом значении в API тоже подставляется 08:00)
- **Исправление:** после срабатывания по расписанию операция появляется в списке **и** баланс счёта обновляется (раньше баланс мог оставаться прежним — с v1.2.2 баланс хранится в базе и требует явного пересчёта при автосоздании операции)

## Кредиты — автосписание

- **Исправление:** операция автосписания сохраняется с заданным временем списания (например, 10:00), а не с 00:00
- В **графике платежей** и **списке кредитов** дата следующего платежа показывается с временем автосписания (`debit_time_local`), а не полуночью
