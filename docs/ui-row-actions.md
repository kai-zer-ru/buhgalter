# UI — действия в строках и фильтры

Единые компоненты для контекстных действий и панелей фильтров.

Связанные документы: [ui-stable-layout.md](ui-stable-layout.md), [ui-navigation.md](ui-navigation.md), [transactions-display.md](transactions-display.md).

## RowActionsMenu

| Файл | Назначение |
|------|------------|
| `$lib/components/RowActionsMenu.svelte` | Кнопка «⋯» и выпадающее меню с пунктами (иконка + подпись) |
| `$lib/components/IconGlyph.svelte` | SVG-иконки для меню и `IconButton`; для дохода / расхода / перевода — глифы Font Awesome Solid (`f067`, `f068`, `f0ec`) |

### NewTransactionButtons

| Файл | Назначение |
|------|------------|
| `$lib/components/NewTransactionButtons.svelte` | Три кнопки-иконки: доход (primary), расход, перевод — в шапке главной, `/transactions`, страницы счёта |

Тип новой операции задаётся кнопкой; в `TransactionForm` переключатель «Доход / Расход» не показывается.

### Правила

1. **Один триггер на строку/блок** — вертикальное меню «⋯» (`more-vertical`), не горизонтальный ряд иконок.
2. **Подписи обязательны** — каждый пункт: иконка слева и текст (`label`); опасные действия — `variant: 'danger'`.
3. **Исключения** — отдельная кнопка допустима, если это главное действие экрана (на странице кредита **«Оплатить»** в шапке слева от меню; в строках графика «Оплатить» остаётся в меню).
4. **Пустое меню** — компонент не рендерится, если нет доступных пунктов.

### Где применено

| Место | Пункты меню |
|-------|-------------|
| `TransactionList` | Повторить, сделать периодической, изменить, удалить (главная, `/transactions`, страница счёта) |
| `/recurring-operations` | Изменить, удалить |
| `CategoriesTab` | Изменить, сделать главной, удалить (категория / подкатегория — без «главной») |
| `DebtList` | Закрыть долг, удалить |
| `/accounts` (список) | Редактировать, сделать основным, архивировать, удалить |
| `/accounts/[id]` (шапка карточки) | **≥ md:** кнопки Доход / Расход / Перевод + меню «⋯» (редактирование, основной, архив, удаление). **&lt; md:** всё в меню «⋯» (сначала операции, затем счёт) |
| `/credits/[id]` (шапка) | Изменить название, счёт, время списания, банк (иконка `bank`), завершить, удалить |
| `/credits/[id]` (строка графика) | Оплатить или удалить платёж |

**Список счетов `/accounts`:** меню «⋯» у каждой карточки (включая основной). «Редактировать» — inline-форма на карточке (подписи: название, банк, начальный баланс). «Сделать основным» скрыт для уже основного счёта. Архив и удаление обновляют список без перезагрузки; удаление — через `confirm` ([ui-dialogs.md](ui-dialogs.md)).

**Настройки → Категории:** «Сделать главной» — в меню «⋯»; главная отмечена галочкой у названия. Раскрытие подкатегорий — клик по «Название ▶» (треугольник после текста). Иконка категории декоративная, не кликабельна.

Save/Cancel в режиме inline-редактирования категорий и форм — **не** в `RowActionsMenu`.

### Пример

```svelte
<RowActionsMenu
	actions={[
		{ icon: 'edit', label: $_('common.edit'), onclick: () => startEdit(item) },
		{
			icon: 'delete',
			label: $_('common.delete'),
			variant: 'danger',
			onclick: () => void remove(item)
		}
	]}
/>
```

## FilterPanel

| Файл | Назначение |
|------|------------|
| `$lib/components/FilterPanel.svelte` | Обёртка `<details>` для фильтров: спойлер на мобильных, всегда открыто на десктопе |
| `$lib/components/TransactionFilters.svelte` | Поля фильтров операций внутри `FilterPanel` |

### Поведение

| Экран | Мобильный (`< md`) | Десктоп (`≥ md`) |
|-------|-------------------|------------------|
| Заголовок | «Фильтры» + chevron ▾ (поворот при раскрытии) | скрыт |
| Содержимое | по умолчанию свёрнуто | всегда видно (`matchMedia` держит `details.open`) |

Стили: `.filter-panel`, `.filter-panel-summary`, `.filter-panel-chevron` в `web/src/routes/layout.css`.

### Где применено

- `/transactions` — через `TransactionFilters`
- `/accounts/[id]` — через `TransactionFilters`
- `/stats` — собственная сетка полей внутри `FilterPanel`

## Чеклист для новых экранов

1. Строковые действия (2+ или второстепенные) — `RowActionsMenu`, не ряд `IconButton`.
2. Главное действие экрана (если одно и частое) — отдельная кнопка рядом с меню, как «Оплатить» на кредите.
3. Блок фильтров на узком экране — `FilterPanel`, не всегда развёрнутая карточка.
4. i18n: `common.actions` — aria-label триггера меню; `transactions.filters.toggle` — заголовок спойлера фильтров.
