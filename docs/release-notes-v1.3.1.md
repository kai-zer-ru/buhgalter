## Навигация

В верхнем меню появился пункт **«Операции»** — быстрый переход на страницу всех операций (`/transactions`). На мобильных — в гамбургер-меню.

## Список операций

На **главной**, в **«Все операции»** и на **странице счёта** списки приведены к одному виду:

| Колонка | Содержимое |
|---------|------------|
| Дата-время | без секунд |
| Счёт | направление перевода «откуда → куда» |
| Категория | иконка категории → иконка подкатегории (если есть) |
| Сумма | с префиксом `+` / `−` где нужно |
| Описание | |
| Действия | меню «⋯» |

На странице **«Все операции»** плановые и прошлые операции разделены **двумя спойлерами** (как на главной): сверху **«Плановые»** (свёрнуты), ниже **«Прошлые операции»** (открыты). Пагинация — только у прошлых.

На странице **должника** блок переименован в **«Последние операции»** и использует тот же компонент (без колонки «Категория»; удаление — по правилам долгов).

## Счета на главной и в списке

Счета сгруппированы в два блока:

1. **«Мои средства»** — наличные и банковские в одной сетке (сначала наличные, затем банки).
2. **«Кредитные средства»** — только кредитные карты.

На **главной** оба блока — в спойлерах (на мобильных свёрнуты по умолчанию). На странице **«Счета»** — те же группы с заголовками, без спойлеров.

## Автопополнение

На карточке банковского счёта вместо «Автопополнение: ✓» показывается **имя счёта списания**, например: `Автопополнение: Яндекс`. Если автопополнение выключено — строка **не отображается**.

Подробнее — [ui-balance-maintenance.md](ui-balance-maintenance.md).

## Категория «Перевод»

**«Перевод»** стала **системной** категорией расходов (иконка — двойная стрелка, как у кнопки перевода). В неё попадают только переводы между **своими** счетами; вручную создать доход/расход в этой категории нельзя.

Если у вас уже были операции в старой пользовательской «Перевод», не связанные с переводами между счетами, они перенесены в обычную категорию **«Переводы»** (создаётся при обновлении, если нужно). Подкатегории старой «Перевод» тоже переносятся в «Переводы».

Подробнее — [categories-and-icons.md](categories-and-icons.md).

## Формы и списки

- В выпадающих списках **счетов**, **категорий** и **подкатегорий** — иконки слева от названия.
- В форме дохода/расхода поле **«Новая подкатегория»** скрывается, если подкатегория уже выбрана в списке.
- В каталоге иконок категорий добавлена **«Караоке»** (микрофон).

## Бюджет

Перестроена вёрстка **формы создания и редактирования** бюджета: поля выстраиваются в строки в зависимости от области (все расходы / категория / подкатегория); на мобильных — больше строк, на десктопе — компактнее. Логика лимитов не изменилась.

Подробнее — [ui-budget.md](ui-budget.md).

## Исправления

- В окне **«Изменить операцию»** календарь больше не уходит за нижний край экрана — при нехватке места открывается **вверх**.

## Для разработчиков

- Общий список операций — `TransactionList.svelte`, колонка категории — `TransactionCategoryCell.svelte`.
- Иконки в селектах — `$lib/select-options.ts`, `SelectOptionIcon.svelte`.
- Миграция категории «Перевод» — `categoryseed.EnsureTransferCategory` при старте приложения (без отдельной SQL-миграции).
