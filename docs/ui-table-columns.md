# UI — порядок колонок в таблицах

Единые правила для списков операций, долгов и смежных экранов.

## Операции (транзакции)

**Порядок:** дата → **счёт** → категория (если есть) → сумма → описание → действия.

| Экран | Колонки |
|-------|---------|
| `/`, `/transactions`, `/accounts/[id]`, `/stats` (drill-down) | Дата, Счёт, Категория, Сумма, Описание, Действия |
| «Последние операции» на `/debtors/[id]` | Дата, Счёт, Сумма, Описание, Удаление (где `deletable`) |

Колонка **«Счёт»** — вторая после даты (или первой контекстной колонки). См. [transactions-display.md](transactions-display.md).

## Долги

**Порядок:** контекст (должник / направление) → **счёт** → сумма → даты → действия.

Колонка **«Счёт»** стоит **сразу перед «Сумма»**, как «Счёт» перед «Сумма» в таблицах операций.

| Экран | Колонки |
|-------|---------|
| `/debts` | Должник, Направление, **Счёт**, Сумма, Операция, План. возврат (`title` → полное название), действия |
| `/debtors/[id]` (список долгов) | Направление, **Счёт**, Сумма, Операция, План. возврат, действия |

Имя счёта — ссылка на `/accounts/{id}`; при «без баланса» — «—».

**Вёрстка:** сумма, даты и кнопки — `whitespace-nowrap`; сумма — `text-right` + `tabular-nums`; контейнер — `overflow-x-auto`, на `<table>` — `min-w-*` при широкой сетке колонок. Заголовок «Планируемая дата возврата» в UI сокращён до `debts.col.dueShort`, полный текст — в `title` и `debts.col.due`.

## Новые таблицы

1. Не ставить «Счёт» в конец строки, если в том же разделе или на соседнем экране есть таблица операций с счётом в начале.
2. Для любых таблиц с датой и счётом: **дата → счёт → … → сумма**.
3. Для таблиц долгов без отдельной колонки «дата операции» в начале: **направление (и должник) → счёт → сумма → даты**.
4. Длинные заголовки колонок — короткая подпись в `<th>`, полный текст в `title` / i18n.
