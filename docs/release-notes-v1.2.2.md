## Обновления

- Приложение **раз в сутки** проверяет наличие новой версии на GitHub
- Уведомление показывается **только администраторам**
- Если доступен более новый релиз — модальное окно с напоминанием:
  - сохранить резервную копию базы (`data/buhgalter.db`, каталог `backups/`) **или скачать бекап** в **Настройки → Админка → Бекапы**;
  - прочитать release notes перед обновлением
- Кнопка **«Открыть release notes»** ведёт на страницу релиза на GitHub
- «Понятно» скрывает уведомление для этой версии до выхода следующей

Проверка не блокирует работу при недоступности GitHub.

## Операции (доход, расход, перевод)

На **главной**, в **«Все операции»** и на **странице счёта** в шапке — три кнопки-иконки:

| Кнопка | Действие |
|--------|----------|
| **+** (доход) | Новая операция дохода |
| **−** (расход) | Новая операция расхода |
| **⇄** (перевод) | Перевод между счетами |

Подсказка при наведении — полное название. На странице счёта кнопки рядом с меню «⋯» (редактирование счёта, основной, архив, удаление).

**Форма новой операции:** переключатель «Доход / Расход» убран — тип уже выбран кнопкой; в заголовке окна отображается «Доход» или «Расход». При **редактировании** существующей операции тип по-прежнему показан текстом и не меняется.

Подробнее — [transactions-display.md](transactions-display.md), [ui-row-actions.md](ui-row-actions.md).

## Статистика

- Блок **«По категориям»** разделён на **доходы** и **расходы**
- При фильтре по типу показывается только соответствующая секция
- Процент категории считается от суммы категорий **того же типа** (а не от общего оборота)
