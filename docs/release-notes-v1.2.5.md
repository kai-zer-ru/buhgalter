# Release notes — v1.2.5

Краткая сводка для пользователей и администраторов. Полный список изменений — [CHANGELOG.md](../CHANGELOG.md).

## In-app уведомления (toast)

Краткие сообщения об успехе, ошибке и предупреждении показываются **внизу справа** (на узком экране — по центру снизу).

- Сохранение, удаление, ошибки API и валидация при отправке формы — через toast
- Постоянные баннеры (например, запрос сброса пароля у админа) и подсказки у полей — без изменений
- Не затрагивает уведомления в **Telegram** и **MAX** — это по-прежнему отдельная настройка

Подробности для разработчиков: [ui-toast.md](ui-toast.md).

## Верхнее меню и настройки

Шапка приложения перестроена:

| Элемент | Содержимое |
|---------|------------|
| Плоские пункты | Счета, Долги, Кредиты, Статистика |
| **Настройки ▼** | Профиль, Пароль, API-токены, Уведомления, Категории, Импорт/экспорт, Периодические операции |
| **Админка ▼** | Только для администратора: Система, Пользователи, Бэкапы, Диагностика |

- На главную — клик по **названию приложения** в шапке (отдельного пункта «Главная» в меню нет; в хлебных крошках «Главная» сохранена)
- На мобильных — гамбургер-меню с теми же группами
- Настройки и админка — отдельные страницы с собственными URL (`/settings/password`, `/admin/users` и т.д.)

### Новые адреса (важно при закладках)

| Было | Стало |
|------|-------|
| `/settings?tab=password` | `/settings/password` |
| `/settings?tab=tokens` | `/settings/tokens` |
| `/import` | `/settings/import` |
| `/recurring-operations` | `/settings/recurring-operations` |
| Админка через вкладку в настройках | `/admin`, `/admin/users`, … |

Старые URL с `?tab=` и прежние `/import`, `/recurring-operations` **не работают** — обновите закладки.

Подробнее — [ui-navigation.md](ui-navigation.md).

## Главная — плановые операции

В спойлере **«Плановые»** операции отсортированы **от новых к старым** (как в спойлере прошлых операций).

## API-токены — срок действия

При создании токена с явной датой истечения в календаре **нельзя выбрать сегодня и прошлые дни** — только завтра и далее (в вашем часовом поясе). Исправлен баг, когда можно было создать токен с датой в прошлом.

Бессрочный токен и срок по умолчанию (30 дней) — без изменений. См. [api/authentication.md](api/authentication.md).
