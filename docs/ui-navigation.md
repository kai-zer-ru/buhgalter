# UI — навигация «Назад» и хлебные крошки

На вложенных экранах (деталь, форма, подраздел настроек) путь от главной до текущей страницы оформляется единообразно.

## Компоненты

| Файл | Назначение |
|------|------------|
| `$lib/components/BackLink.svelte` | Хлебные крошки — цепочка ссылок от «Главная» до текущего экрана |
| `$lib/components/NavDropdown.svelte` | Выпадающее подменю в шапке (десктоп) и accordion-группа (мобилка) |

## Правила

1. **Всегда сверху** — первый элемент контента страницы (до заголовка `h1` и основного содержимого).
2. **Не внизу** — не дублировать ссылку «назад» под таблицами и формами.
3. **Цепочка от главной** — первый элемент всегда «Главная» (`nav.home`, `href: '/'`); далее разделы и подразделы; последний элемент — текущая страница (без ссылки).
4. **Родительские маршруты** — промежуточные элементы ведут на логического «родителя»:
   - `/accounts/new`, `/accounts/[id]` → `/accounts`
   - `/transactions` → `/` (главная) + «Все операции»
   - `/settings/password`, … → `/` + «Настройки» + подраздел
   - `/admin/users` → `/` + «Админка» + «Пользователи»
   - `/debtors/[id]` → `/debts`
   - `/credits/[id]` → `/credits`
5. **Подписи** — `label` из i18n (заголовок раздела).
6. **Один URL на страницу** — без query-вкладок (`?tab=`) и без редиректов со старых адресов.

## Пример

```svelte
<script lang="ts">
	import BackLink from '$lib/components/BackLink.svelte';
	import { _ } from 'svelte-i18n';
</script>

<div class="space-y-6">
	<BackLink
		items={[
			{ href: '/', label: $_('nav.home') },
			{ href: '/accounts', label: $_('accounts.title') },
			{ href: '/accounts/new', label: $_('accounts.new') }
		]}
	/>

	<h1 class="text-2xl font-semibold">…</h1>
	<!-- остальной контент -->
</div>
```

Допустимые значения `href` — тип `BackLinkHref` в `BackLink.svelte`. При новом вложенном экране добавьте маршрут в этот тип, если родителя там ещё нет.

## Где применено

| Маршрут | Цепочка |
|---------|---------|
| `/accounts` | Главная → Счета |
| `/accounts/new`, `/accounts/[id]` | Главная → Счета → … |
| `/transactions` | Главная → Все операции |
| `/debts`, `/credits`, `/stats`, `/budget` | Главная → раздел |
| `/debtors/[id]` | Главная → Долги → должник |
| `/credits/[id]` | Главная → Кредиты → кредит |
| `/settings` | Главная → Настройки |
| `/settings/password`, … | Главная → Настройки → подраздел |
| `/admin`, `/admin/users`, … | Главная → Админка (+ подраздел) |

Badge-метки и спойлеры графика на `/credits` — [ui-credits.md](ui-credits.md).

### Ширина контента (v1.3)

Шапка и `<main>` — `max-w-5xl` (`80rem`, `--container-5xl` в `layout.css`); на узких экранах — полная ширина с отступами `px-4` / `sm:px-6`.

Меню «⋯» в строках и спойлер фильтров — [ui-row-actions.md](ui-row-actions.md).

## Требование для новых экранов

При добавлении страницы второго уровня (деталь, создание, подраздел):

1. Вставить `<BackLink items={[…]}>` **в начале** разметки; первый элемент — `{ href: '/', label: $_('nav.home') }`.
2. Не использовать текстовые ссылки «← …» внизу или вместо крошек.
3. Главная навигация (`nav` в layout) остаётся отдельно — `BackLink` только для контекста текущего раздела.

Чеклист навигации — в этом файле (разделы «Кликабельные сущности», «Где применить»).

## Вкладки внутри страницы

На страницах с **контентными** подвкладками (не навигация раздела) полоса `.tab` остаётся:

| Маршрут | Подвкладки |
|---------|------------|
| `/settings/categories` | Расходы / Доходы |
| `/settings/import` | Импорт / Экспорт |
| `/accounts` | Активные / Архивные / Удалённые |
| `/debts` | Активные / Закрытые |
| `/debtors/[id]` (секция долгов) | Активные / Закрытые |
| `/credits` | Активные / Завершённые |

`PageTabs` — `$lib/components/PageTabs.svelte`.

### `/settings/recurring-operations` (v1.2)

- Сначала отображается таблица периодических операций.
- Ниже — кнопка «Добавить», раскрывающая форму создания в этом же экране.
- Нажатие «Изменить» открывает inline-форму редактирования внутри блока списка.
- Одновременно открыта только одна форма редактирования; при открытии другой предыдущая закрывается.
- Время выполнения по умолчанию — **08:00** в часовом поясе пользователя (поле в блоке «Время выполнения»); в API пустое `time_local` трактуется так же.
- Планировщик backend создаёт операции по `next_run_at` и **пересчитывает баланс** затронутых счетов (`accounts.current_balance`).

### `/budget` (v1.3)

- Переключатель месяца (`?month=YYYY-MM`), карточки с прогресс-баром.
- CRUD: inline-форма, `RowActionsMenu` — см. [ui-budget.md](ui-budget.md).

## Кликабельные сущности

Имена связанных сущностей в списках, таблицах, виджетах и карточках — **ссылки** на соответствующую страницу. Обычный текст без ссылки недопустим, если сущность имеет детальный экран.

| Сущность | Куда ведёт | Примеры мест |
|----------|------------|--------------|
| Счёт (кошелёк) | `/accounts/{id}` | операции, переводы, dashboard, кредиты, **списки долгов**, статистика |
| Кредит | `/credits/{id}` | график платежей, операции, уведомления, статистика |
| Должник | `/debtors/{id}` | список долгов, операции по человеку, виджет на главной |

### Правила

1. **Везде одинаково** — одно и то же имя счёта/кредита/должника кликабельно на всех экранах, где оно показывается (не только на «родной» странице раздела).
2. **Стиль** — ссылка в цвете `--primary`, без подчёркивания по умолчанию; `hover:underline` или лёгкое изменение opacity — как у остальных in-app ссылок.
3. **Не перехватывать клик** — если строка таблицы целиком кликабельна, вложенная ссылка на другую сущность должна работать (`stopPropagation` при необходимости).
4. **Должник** — страница `/debtors/{id}`: сводка, долги и операции; кнопки «Дать в долг ещё» / «Взять в долг ещё» — **только по направлению активных долгов** (нельзя «взять» у того, кому уже «дали», и наоборот). Операции долга после погашения удаляются только вместе с долгом (см. [ui-dialogs.md](ui-dialogs.md#где-уже-применено)).
5. **Поиск и drill-down** — те же правила кликабельности обязательны для результатов поиска и переходов drill-down (в т.ч. `/stats`): без «текстовых» имён сущностей.

### Где применить

| Раздел | Что |
|------|-----|
| Счета | список `/accounts`: два блока с заголовками (без спойлеров) — [ui-credit-cards.md](ui-credit-cards.md); клик по карточке → `/accounts/{id}`; меню «⋯» на карточке — [ui-row-actions.md](ui-row-actions.md) |
| Долги | должник → `/debtors/{id}`, страница `/debtors/[id]`, порядок колонок — [ui-table-columns.md](ui-table-columns.md); дата операции — optional (время свёрнуто), плановый возврат — только дата — [date-time-display.md](date-time-display.md) |
| Кредиты | кредит → `/credits/{id}` |
| Статистика | счёт, кредит, должник в поиске и drill-down |
| Главная и операции | счёт на главной, `/transactions`, `/accounts/[id]` |
