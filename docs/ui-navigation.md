# UI — навигация «Назад»

На вложенных экранах (деталь, форма, подраздел настроек) кнопка возврата к родительской странице оформляется единообразно.

## Компонент

| Файл | Назначение |
|------|------------|
| `$lib/components/BackLink.svelte` | Кнопка «назад» — ссылка на родительский маршрут |

## Правила

1. **Всегда сверху** — первый элемент контента страницы (до заголовка `h1` и основного содержимого).
2. **Не внизу** — не дублировать ссылку «назад» под таблицами и формами.
3. **Кнопка, не текст** — использовать `BackLink`, а не обычную текстовую ссылку (`← Назад`, `style:color="var(--primary)"` и т.п.).
4. **Родительский маршрут** — `href` указывает на логического «родителя»:
   - `/accounts/new`, `/accounts/[id]` → `/accounts`
   - `/transactions` → `/` (главная)
   - `/settings/categories` → `/settings`
   - `/debtors/[id]` → `/debts`
   - `/credits/[id]` → `/credits`
5. **Подпись** — `label` = заголовок родительской страницы (i18n), например `accounts.title`, `dashboard.title`, `settings.title`.

## Пример

```svelte
<script lang="ts">
	import BackLink from '$lib/components/BackLink.svelte';
	import { _ } from 'svelte-i18n';
</script>

<div class="space-y-6">
	<BackLink href="/accounts" label={$_('accounts.title')} />

	<h1 class="text-2xl font-semibold">…</h1>
	<!-- остальной контент -->
</div>
```

Допустимые значения `href` — тип `BackLinkHref` в `BackLink.svelte` (в т.ч. `'/'`, `'/accounts'`, `'/settings'`, `'/credits'`, `'/recurring-operations'`). При новом вложенном экране добавьте маршрут в этот тип, если родителя там ещё нет.

## Где применено

- `accounts/new`, `accounts/[id]` → счета
- `transactions` → главная
- `settings/categories` → настройки
- `debtors/[id]` → долги
- `credits/[id]` → кредиты

Badge-метки и спойлеры графика на `/credits` — [ui-credits.md](ui-credits.md).

## Мобильная навигация (v1.1)

**Файл:** `web/src/routes/+layout.svelte`

| Элемент | Поведение |
|---------|-----------|
| Breakpoint | Меню-гамбургер — `sm:hidden`; десктопная `nav` — `hidden sm:flex` |
| Пункты | Главная, Счета, Долги, Кредиты, Периодические операции, Статистика, Настройки |
| Активный пункт | `.nav-link-active` по текущему пути (`+layout.svelte`); `aria-current="page"` |
| Закрытие | `afterNavigate` сбрасывает открытое меню; backdrop по тапу вне меню |
| Z-index | Шапка `sticky top-0 z-50`; меню поверх контента страницы |

Touch-targets и карточки таблиц — [ui-stable-layout.md](ui-stable-layout.md), `web/src/routes/layout.css` (`@media (pointer: coarse)`).

Меню «⋯» в строках и спойлер фильтров — [ui-row-actions.md](ui-row-actions.md).

## Требование для новых экранов

При добавлении страницы второго уровня (деталь, создание, подраздел):

1. Вставить `<BackLink>` **в начале** разметки.
2. Не использовать текстовые ссылки «← …» внизу или вместо кнопки.
3. Главная навигация (`nav` в layout) остаётся отдельно — `BackLink` только для возврата в контексте текущего раздела.

Чеклист навигации — в этом файле (разделы «Кликабельные сущности», «Где применить»).

## Вкладки раздела

Единый каркас: **один `h1`**, полоса вкладок (`.tab` / `.tab-active`), контент ниже. Между вкладками одного раздела **нет** `BackLink`.

| Компонент | Файл |
|-----------|------|
| `PageTabs` | `$lib/components/PageTabs.svelte` — кнопки вкладок |
| `SectionHeader` | `$lib/components/SectionHeader.svelte` — `h1` + слот действий |

### Где применено

| Маршрут | Вкладки | Подвкладки |
|---------|---------|------------|
| `/settings?tab=` | Профиль, Пароль, Токены, Уведомления, Категории, Импорт | Категории: расходы/доходы; Импорт: импорт/экспорт |
| `/accounts` | Активные / Архив | — |
| `/debts` | Активные / Закрытые | — |
| `/debtors/[id]` (секция долгов) | Активные / Закрытые | — |
| `/credits` | Активные / Завершённые | — |
| `/settings?tab=admin&section=` | Админка как раздел внутри настроек (для admin) | system/users/backups/diagnostics |

`BackLink` — только при уходе **из** раздела (деталь → список), не между соседними вкладками.

### `/recurring-operations` (v1.2)

- Сначала отображается таблица периодических операций.
- Ниже — кнопка «Добавить», раскрывающая форму создания в этом же экране.
- Нажатие «Изменить» открывает inline-форму редактирования внутри блока списка.
- Одновременно открыта только одна форма редактирования; при открытии другой предыдущая закрывается.

## Кликабельные сущности

Имена связанных сущностей в списках, таблицах, виджетах и карточках — **ссылки** на соответствующую страницу. Обычный текст без ссылки недопустим, если сущность имеет детальный экран.

| Сущность | Куда ведёт | Примеры мест |
|----------|------------|--------------|
| Счёт (кошелёк) | `/accounts/{id}` | операции, переводы, dashboard, кредиты, **списки долгов**, статистика |
| Кредит | `/credits/{id}` | график платежей, операции, уведомления, статистика |
| Должник | `/debtors/{id}` | список долгов, операции по человеку, виджет на главной |

### Правила

1. **Везде одинаково** — одно и то же имя счёта/кредита/должника кликабельно на всех экранах, где оно показывается (не только на «родной» странице раздела).
2. **Стиль** — ссылка в цвете `--primary`, без подчёркивания по умолчанию; `hover:underline` или лёгкое изменение opacity — как у остальных in-app ссылок.
3. **Не перехватывать клик** — если строка таблицы целиком кликабельна, вложенная ссылка на другую сущность должна работать (`stopPropagation` при необходимости).
4. **Должник** — страница `/debtors/{id}`: сводка, долги и операции; кнопки «Дать в долг ещё» / «Взять в долг ещё» — **только по направлению активных долгов** (нельзя «взять» у того, кому уже «дали», и наоборот). Операции долга после погашения удаляются только вместе с долгом (см. [ui-dialogs.md](ui-dialogs.md#где-уже-применено-этапы-15)).
5. **Поиск и drill-down** — те же правила кликабельности обязательны для результатов поиска и переходов drill-down (в т.ч. `/stats`): без «текстовых» имён сущностей.

### Где применить

| Раздел | Что |
|------|-----|
| Счета | список `/accounts`: клик по карточке → `/accounts/{id}`; меню «⋯» на карточке — [ui-row-actions.md](ui-row-actions.md) |
| Долги | должник → `/debtors/{id}`, страница `/debtors/[id]`, порядок колонок — [ui-table-columns.md](ui-table-columns.md); даты в форме и списках — **без времени** (см. [ui-dialogs.md](ui-dialogs.md#datetimepicker-и-всплывающие-панели)) |
| Кредиты | кредит → `/credits/{id}` |
| Статистика | счёт, кредит, должник в поиске и drill-down |
| Главная и операции | счёт на главной, `/transactions`, `/accounts/[id]` |
