## Сброс пароля (через администратора)

Self-service по e-mail **не** реализован — сценарий для self-hosted:

1. Пользователь на `/login` → **«Запросить сброс пароля»** (логин).
2. Администратор видит баннер и список запросов; переходит в **Админка → Пользователи**.
3. Админ задаёт новый пароль (`PUT /api/v1/admin/users/{id}/password`); сессии пользователя сбрасываются.

API:

| Метод | Путь | Кто |
|-------|------|-----|
| `POST` | `/api/v1/auth/request-password-reset` | любой (rate limit по IP) |
| `GET` | `/api/v1/admin/password-reset-requests` | админ |
| `POST` | `/api/v1/admin/password-reset-requests/{id}/ack` | админ |
| `PUT` | `/api/v1/admin/users/{id}/password` | админ |

Миграция: `023_password_reset_requests.sql`.

Подробнее: [api/authentication.md](api/authentication.md).

## Кредиты

### График с процентами

- Ровно `term_months` платежей (исправлен укороченный график, например 22 вместо 36).
- Платежи 1…N−1 — равный аннуитет; последний закрывает остаток тела и проценты за период.
- При старте сервера активные кредиты с неполным графиком **дополняются** (`RepairShortSchedules`, миграция-маркер `020`).

### Редактирование графика

- На странице кредита → блок «Неоплаченные» → изменить суммы будущих `scheduled`-платежей.
- `PATCH /api/v1/credits/{id}/schedule` — только неоплаченные строки; связанные операции `kind=future` обновляются.

### Кредит задним числом

- Флаг **«Уже платил по графику»** при создании — прошлые платежи учитываются в `paid_amount` без влияния на баланс (по умолчанию).
- **«Списан со счёта»** — для хвоста последних прошлых платежей (`retroactive_debit_count`): создаётся расход на счёте списания, платёж попадает в статистику.

Подробнее: [ui-credits.md](ui-credits.md).

## Переводы и категории

- При переводе между счетами — опциональное поле **комиссии**; сумма уходит расходом в системную категорию **«Комиссия»** (иконка `percent`).
- Системные категории: «Комиссия», «Кредиты», «Долги» (income/expense) — seed при создании пользователя и backfill при старте.
- При ручной сортировке категорий системные всегда в конце; системную нельзя сделать главной.

Подробнее: [categories-and-icons.md](categories-and-icons.md), [transactions-display.md](transactions-display.md).

## Обновление с v1.0.0

1. Сделайте бэкап БД ([README — Бэкапы](../README.md#бэкапы)).
2. Замените бинарник или образ Docker (`ghcr.io/kai-zer-ru/buhgalter:v1.1.0` / `latest`).
3. При старте применятся миграции `020`, `023`; для существующих пользователей создастся категория «Комиссия», при необходимости дополнятся графики кредитов.

Версию можно проверить в **Админка → Диагностика**.
