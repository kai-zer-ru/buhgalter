# UI — статистика

Страница `/stats`: сводка, разбивка по периодам и категориям, поиск по операциям.

Связанные документы: [ui-row-actions.md](ui-row-actions.md), [transactions-display.md](transactions-display.md), [ui-navigation.md](ui-navigation.md), [api/openapi.yaml](api/openapi.yaml).

---

## Структура экрана

| Блок | Источник API | Описание |
|------|--------------|----------|
| Сводка | `GET /stats/summary` | Доход, расход, дельта, число операций |
| По периодам | `GET /stats/by-period` | Группировка `day` / `week` / `month`; человекочитаемые подписи; от новых к старым; на мобильных — мини-бары |
| По категориям | `GET /stats/by-category` | Две секции — **доходы** и **расходы**; сумма, % от оборота своего типа, число операций |
| Поиск | `GET /stats/search` | Появляется при непустом поле поиска; список операций (`TransactionList`) |

В блоке **«По категориям»** категории разделены на **доходы** и **расходы** (при фильтре «тип» показывается только выбранная секция). Отдельная таблица по подкатегориям в UI **не выводится** (endpoint `GET /stats/by-subcategory` в API сохранён для интеграций).

## Фильтры

Панель в `FilterPanel` (`web/src/routes/stats/+page.svelte`):

| Поле | Query-параметр | Примечание |
|------|----------------|------------|
| С | `from` | Дата без времени; граница суток в timezone пользователя |
| По | `to` | То же |
| Тип | `type` | `income` / `expense` / все |
| Счёт | `account_id` | |
| Категория | `category_id` | Список фильтруется по выбранному типу |
| Группировка | `group_by` | Только для блока «По периодам» |
| Поиск | `q` (в `search`) | Только для блока результатов поиска |

Фильтры применяются автоматически при изменении (`$effect`). На мобильных панель свёрнута под спойлер «Фильтры» — см. [ui-row-actions.md](ui-row-actions.md).

## Блок «По периодам»

### Подписи

API возвращает технический ключ `period` (`YYYY-MM-DD` в timezone пользователя). В UI подпись форматируется по выбранной группировке (`$lib/stats-period.ts`, локаль интерфейса):

| `group_by` | Пример (ru) | Примечание |
|------------|-------------|------------|
| `day` | `2026-06-15` | ISO-дата без изменений |
| `week` | `1–7 июня 2026` | Неделя с понедельника; при переходе через месяц — «29 июня – 5 июля 2026» |
| `month` | `Июнь 2026` | Название месяца и год |

### Порядок

Строки отсортированы **от новых к старым** (сначала текущий месяц/неделя/день). Сортировка выполняется на backend при формировании ответа `GET /stats/by-period`.

## Drill-down

Клик по названию категории в таблице → `/transactions` с query:

- `category_id`
- `from_local`, `to_local` (даты из фильтров)
- при необходимости `type`, `account_id`

Категории с одинаковым именем (например «Долги») показываются с суффиксом **(Доход)** / **(Расход)** — `$lib/category-label.ts`.

В результатах поиска имена должников и кредитов — кликабельные ссылки (`EntityLink`), как на других экранах — [ui-navigation.md](ui-navigation.md).

## API

Полное описание параметров и схем — в OpenAPI, тег `/stats/*`:

| Endpoint | Назначение |
|----------|------------|
| `GET /stats/summary` | Сводка |
| `GET /stats/by-category` | Агрегация по категориям |
| `GET /stats/by-period` | Агрегация по периодам (`group_by`) |
| `GET /stats/search` | Список операций по подстроке в описании |
| `GET /stats/context` | Контекстная сводка (счёт, должник, кредит, долги) |
| `GET /stats/by-subcategory` | Агрегация по подкатегориям (не используется веб-UI) |

Суммы — в копейках (`integer`), как в остальном API.
