# UI — кеш справочников в веб-клиенте

In-memory кеш GET-ответов для редко меняющихся справочников. Уменьшает число HTTP-запросов при переходах между страницами (формы операций, фильтры, статистика и т.д.).

Связанные документы: [api/openapi.yaml](api/openapi.yaml), [categories-and-icons.md](categories-and-icons.md).

---

## Реализация

| Файл | Назначение |
|------|------------|
| `web/src/lib/api/cache.ts` | Хранилище, TTL, дедупликация параллельных запросов, инвалидация по префиксу |
| `web/src/lib/api/client.ts` | Обёртки `cachedGet` для справочников; сброс кеша в мутациях и `logout` |
| `web/src/lib/stores/auth.ts` | Полная очистка кеша при logout и 401 |

## Кешируемые эндпоинты

| Эндпоинт | TTL | Инвалидация |
|----------|-----|-------------|
| `GET /banks` | 5 мин | — (справочник только для чтения) |
| `GET /categories` (с `?type=` или без) | 5 мин | POST/PUT/DELETE категорий и подкатегорий, reorder, set primary |
| `GET /debtors` | 5 мин | `POST /debtors` |

Параллельные вызовы одного и того же ключа (например два компонента одновременно запрашивают `listCategories('expense')`) объединяются в один HTTP-запрос.

## Намеренно без кеша

| Эндпоинт | Причина |
|----------|---------|
| `GET /accounts` | В ответе баланс — устареет после операций |
| `GET /dashboard`, `GET /transactions`, `/stats/*` | Должны отражать актуальное состояние |
| `GET /credits` | Меняется при платежах |

## Жизненный цикл

1. Первый запрос — HTTP, результат кладётся в `Map` с `expiresAt = now + 5 мин`.
2. Повтор в пределах TTL — ответ из памяти, без сети.
3. Мутация справочника — `invalidateApiCache('/api/v1/categories')` (или аналог для debtors).
4. Logout / 401 — `invalidateApiCache()` без аргумента, полная очистка.

Внешние API-клиенты (curl, мобильное приложение) кеш веб-UI не используют — это поведение только встроенного SvelteKit-интерфейса.
