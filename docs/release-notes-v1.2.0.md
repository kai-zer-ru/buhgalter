# Release notes — v1.2.0

Краткая сводка для пользователей и администраторов. Полный список изменений — [CHANGELOG.md](../CHANGELOG.md).

---
## Интерфейс

### UI/UX

- Календарь выровнен по высоте для всех месяцев (без скачков сетки)
- Вместо одноуровневого back-link внедрены хлебные крошки на страницах разделов
- На длинных графиках платежей по кредиту добавлена пагинация (вместо кнопки раскрытия/«показать ещё»)
- Для создания кредита длинный предварительный график также отображается с пагинацией
- Обновлены кнопки и верхнее меню:
  - у nav-кнопок убран border
  - у остальных кнопок добавлен тонкий border
  - активный пункт меню выделяется цветом текста
- Активные вкладки переведены на underline без фоновой заливки
- В настройках уведомлений блок триггеров отображается вертикальным списком
- Админ-раздел перенесён в «Настройки → Админка»
- На главной и на странице счёта баланс показывается в двух значениях: текущий и «с учётом планов за текущий месяц» (второе значение скрывается, если оно равно текущему)
- Исправлен регресс с календарём: компонент `DateTimePicker` больше не меняет высоту между месяцами и состояниями поля
- Функция «Сделать счёт основным» восстановлена на странице счёта и добавлена на список счетов; основной счёт отмечается синей галочкой рядом с названием

### Периодические операции

- Добавлена отдельная страница периодических операций
- Поддержаны периоды: неделя, 2 недели, месяц, год
- Операцию можно создать из существующей транзакции («Сделать периодической») с предзаполнением полей
- Выполнение периодических операций добавлено в планировщик backend
- На странице сначала показывается список, форма добавления раскрывается по кнопке «Добавить»
- Редактирование открывается inline прямо в блоке списка; одновременно открыта только одна форма редактирования
- В селекторе категорий скрыты системные категории (долги/кредиты и др. служебные)
- Поле подкатегории показывается всегда (при пустом списке — disabled)
- Для периода «Раз в месяц» день берётся из даты старта, отдельное поле дня месяца скрыто
- В списке операций для системных категорий скрывается действие «Сделать периодической»

## Кредиты

- В кредите добавлено локальное время автосписания (`HH:MM`)
- Автосписание управляется переключателем: при включении по умолчанию подставляется `00:00`, при выключении автосписание отключается
- На странице кредита добавлено управление автосписанием (переключатель + время)
- В создании кредита переключатель автоплатежей возвращён в форму, а при включении по умолчанию подставляется время `00:00`
- Добавлен банк кредита (`bank_id` + отображение имени банка)
- Смена банка кредита доступна без ограничений
- Автосписание работает по дате/времени через планировщик и не создаёт массово будущие транзакции при создании кредита
- В списке и карточке кредитов отображаются иконки банков
- После оплаты по графику интерфейс сразу отражает изменение и блокирует редактирование графика до ответа сервера
- Сортировка в кредитах приведена к порядку «новые → старые»
- В списках и на странице кредита отображается бейдж типа: «Кредит» или «Рассрочка»
- На странице кредита можно переименовать кредит без перехода в отдельный экран
- Информационный блок кредита перестроен в 3 колонки
- В графике платежей действия «оплатить/удалить» переведены на иконки и показываются только в корректных состояниях
- Добавлен тип кредита «Ипотека»:
  - поля `property_price` и `down_payment`
  - сумма кредита считается автоматически как `property_price - down_payment`
  - можно включить/выключить списание первоначального взноса с баланса
  - при включённом списании можно выбрать отдельный счёт взноса (`down_payment_account_id`)
  - для ипотеки в MVP используется только ежемесячный график (`month`), селектор периодичности в форме скрыт
- Денежные суммы в кредитах унифицированы: разделители тысяч и символ валюты отображаются единообразно в списке, карточке и графике платежей

## Операции и правила планирования

- Для системных категорий (например, кредиты и долги) запрещено создавать/обновлять операции как плановые (`kind=future`)
- В API для такого случая возвращается ошибка валидации с ключом `ERR_SYSTEM_CATEGORY_PLANNED`
- Исправлен регресс: оплата кредита будущей датой снова влияет на прогноз баланса текущего месяца (главная + страница счёта)

## Уведомления и безопасность

- Добавлен триггер уведомлений `password_reset` (только для администраторов)
- Триггер доступен в настройках только у админов (backend + frontend фильтрация)
- При запросе сброса пароля отправляется уведомление всем администраторам с включённым триггером

## Окружение и документация

- Уточнены `BUHGALTER_STATIC_EMBED` и `BUHGALTER_CORS_ORIGINS` в `.env`-документации
- Серверные локали `ru/en` встроены в бинарник; типовой запуск больше не требует `BUHGALTER_LOCALES_DIR` и внешнего каталога локалей
- На `/setup` добавлено восстановление БД из файла `.db` до завершения первичной настройки
- OpenAPI обновлён в двух спецификациях:
  - `docs/api/openapi.yaml`
  - `server/internal/docs/openapi.yaml`
- В OpenAPI добавлен endpoint `POST /api/v1/setup/restore` (multipart `file` + `confirm=RESTORE`)
- В схему `Transaction` добавлено поле `category_is_system`
- В кредитные схемы добавлены ипотечные поля (`credit_kind`, `property_price`, `down_payment`, `down_payment_affects_balance`) и поле `down_payment_account_id` в `CreateCreditRequest`
