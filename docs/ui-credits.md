# UI — кредиты

Единые правила отображения кредитов на `/credits` и `/credits/[id]`: бейджи, график платежей, действия и ограничения.

Связанные документы: [data-model.md](data-model.md), [ui-dialogs.md](ui-dialogs.md), [ui-stable-layout.md](ui-stable-layout.md), [ui-navigation.md](ui-navigation.md).

---

## Badge-метки

Классы в `web/src/routes/layout.css`:

| Класс | Назначение |
|-------|------------|
| `.badge` | Нейтральная метка |
| `.badge-success` | Акцентная метка для завершённого кредита |

Основные метки:

| Метка | i18n | Условие |
|-------|------|---------|
| Рассрочка | `credits.badge.installment` | `interest_rate === 0` |
| Добавлен в учёт после выдачи | `credits.badge.retroactive` | `added_retroactively` |
| Завершённый | `credits.badge.closed` | `status === 'closed'` (на детальной) |

## Список `/credits`

- Вкладки: активные (`status=active`) и завершённые (`status=closed`).
- Сортировка: новые кредиты сверху.
- Под названием кредита показываются бейджи типа/состояния.
- Денежные суммы в списке: единый формат (разделители тысяч + валюта).

## Детальная страница `/credits/[id]`

- Информационный блок кредита отображается в 3 колонки.
- Название кредита можно изменить на странице.
- **Шапка:** кнопка «Оплатить» (если есть ближайший неоплаченный платёж) — слева; меню «⋯» справа (изменить название, счёт, время списания, банк, завершить, удалить).
- **Строки графика:** действия в меню «⋯» (оплатить или удалить платёж) — см. [ui-row-actions.md](ui-row-actions.md).

### Группы графика

График разделён на группы:

| Группа | Критерий |
|--------|----------|
| Неоплаченные | `kind=scheduled`, `is_applied=false` |
| Оплаченные | `kind=scheduled`, `is_applied=true` |
| Учтённые при добавлении | `kind=retroactive` |

Для длинных групп — `TransactionPagination` (по 10 строк). См. [ui-pagination.md](ui-pagination.md).

### Правила действий в графике

| Действие | Правило |
|----------|---------|
| Оплатить | Только ближайший неоплаченный `scheduled` |
| Счёт при оплате | В модалке «Оплата» — поле «Счёт»; по умолчанию основной счёт кредита; другой счёт применяется только к операции данного платежа (`account_id` в API), `debit_account_id` кредита не меняется |
| Удалить оплаченный | Только последний оплаченный `scheduled` |
| Удаление оплаченного | Платёж восстанавливается как неоплаченный `scheduled`, связанная операция удаляется |
| Редактировать сумму | Только для неоплаченных `scheduled` |

### Модалка «Оплата»

| Поле | Поведение |
|------|-----------|
| Сумма | По умолчанию — сумма ближайшего платежа (не больше остатка) |
| Счёт | Активные счета пользователя; по умолчанию `debit_account_id` кредита |
| Дата | Календарь; при открытии из строки графика — дата платежа; кнопки «Сегодня» нет |

Статусы в колонке:
- `credits.payment.status.pending`
- `credits.payment.status.future`
- `credits.payment.status.applied`
- `credits.payment.status.retroactive`
- `credits.payment.status.debitedFromAccount` (для `retroactive` с `transaction_id`)

## Форма кредита (`CreditForm`)

| Элемент | Поведение |
|---------|-----------|
| `credit_kind` | `consumer` / `mortgage` |
| Ипотека | `property_price`, `down_payment`, расчёт суммы `price - down_payment` |
| Взнос ипотеки | Можно не списывать с баланса; при списании можно выбрать отдельный счёт (`down_payment_account_id`) |
| Период для ипотеки | В UI фиксирован `month` |
| Ежемесячный платёж | Автоматически рассчитывается при изменении параметров; поле можно отредактировать вручную. Для ипотеки допускается сумма из договора банка (может отличаться от модели с ежедневными процентами). Если введённая сумма совпадает с расчётной в пределах 1 ₽, override сбрасывается |
| Предпросмотр графика | Запрос `POST /api/v1/credits/schedule/preview` при изменении полей; при ошибке валидации предыдущий график не очищается, показывается `scheduleError` |
| Пагинация графика | 10 строк на страницу; «В начало», «Назад», «Вперёд», «В конец» |
| Автосписание | Локальное время `debit_time_local`, управление переключателем; по умолчанию **08:00** (TZ пользователя); в графике и списке кредитов дата платежа показывается с этим временем; при автосписании операция в БД получает то же время |
| Ретроучёт | `added_retroactively` + хвост списаний через `retroactive_debit_count` |
| Доход по сумме кредита | Только тип «Кредит» (`interest_rate > 0`): переключатель «Учитывать доход в балансе» (`principal_affects_balance`); при включении — `income` на `debit_account_id`. Заблокирован, если в графике есть платёж с датой в прошлом (независимо от «Уже платил по графику» и «Списан со счёта») |

## Интеграционные правила

- Операции, связанные с платежами кредита (`credit_payment_linked`), не редактируются через общий `PUT /transactions/{id}`.
- Для обновления сумм графика используется `PATCH /api/v1/credits/{id}/schedule`.

---

## Чеклист для новых экранов кредитов

1. Использовать `.badge` / `.badge-success` для меток.
2. Для длинного графика использовать пагинацию, не «показать ещё».
3. Не нарушать правила доступности действий (оплата/удаление только в допустимом состоянии).
4. Для destructive-действий использовать общий confirm-паттерн из [ui-dialogs.md](ui-dialogs.md).
