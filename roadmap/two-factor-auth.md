# Двухфакторная аутентификация (2FA)

Планируется в **v1.6.0** ([ROADMAP](../ROADMAP.md#v160)).

## Зачем

Сейчас вход — логин и пароль (сессии / API-токены). Для self-hosted с доступом из интернета нужен второй фактор: даже при утечке пароля без кода из приложения не войти.

У Android уже есть **блокировка приложения** (PIN / биометрия) — это защита экрана телефона, не замена 2FA на сервере.

## Scope (v1.6.0)

| Возможность | Суть |
|-------------|------|
| TOTP | Подключение Google Authenticator / аналогов; секрет + QR при включении |
| Вход | После пароля — одноразовый код |
| Восстановление | Резервные коды при включении; без второго фактора — через уже существующий сброс пароля / админа (уточнить) |
| API-токены | Не ломать существующие токены; опционально позже требовать 2FA для создания токена |
| Web + Android | Оба клиента запрашивают код при логине |

## Не входит в первую очередь

- Вход через организацию (OIDC) — отдельная тема позже.
- SMS-коды (дорого, нестабильно для self-host).
- Passkeys / WebAuthn на сервере — можно после TOTP.
