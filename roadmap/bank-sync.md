# Синхронизация с выпиской банка (РФ)

Планируется в **v1.6.0** ([ROADMAP](../ROADMAP.md#v160)).

## Зачем

Сейчас операции вручную или [импортом Cubux](../docs/import/cubux.md). Нужен путь: **выписка российского банка** → операции в учёте → разнести по категориям ([правила](category-rules-inbox.md)).

## Ориентир

- **Только РФ.** Зарубежные агрегаторы (Plaid, SimpleFIN, GoCardless и т.п.) **не рассматриваем**.
- Реалистичные каналы: файл выписки (CSV/Excel/выписка из приложения банка) → улучшенный импорт; при наличии официального API конкретного банка РФ — плагин позже.
- Скрейпинг ЛК банка — хрупко и сомнительно, не базовый путь.

Смежный канал без файла: [перехват уведомлений на Android](notification-intercept.md).

## Архитектура (черновик)

```
выписка (файл) или connector банка РФ
    → normalize → dedupe → preview/commit (как импорт)
    → правила категорий / очередь «разобрать»
```

- Idempotency — те же идеи, что у импорта ([data-model.md](../docs/data-model.md)); внешний id строки выписки.
- Одна привязка → один `account_id` (банк / кредитка).

## UI

- Загрузка выписки / «синхронизировать» на счёте.
- Список неразнесённых до подтверждения или auto с категорией по правилам.
