# Адаптация под мобильные браузеры

Не входит в v1. Черновик для обсуждения.

## Зачем

Веб-интерфейс ориентирован на desktop; на телефоне неудобны широкие таблицы, горизонтальный скролл и мелкие кнопки. Цель — комфортная работа в мобильном браузере без отдельного нативного клиента ([Android-клиент](android-client.md) — отдельный пункт roadmap).

## Направления

- Компактное меню и навигация, без горизонтального скролла на 320px+
- Таблицы операций — карточный вид или горизонтальный скролл с фиксированными колонками
- Формы и модалки на всю ширину экрана, крупные touch-targets
- Сворачиваемые панели фильтров на узком экране
- Проверка в Chrome DevTools (iPhone SE, Pixel) и на реальном устройстве

## Зависимости

- Стабильный каркас вкладок и единый стиль UI (см. `docs/ui-stable-layout.md`, `docs/ui-table-columns.md`)
- Не дублировать усилия с [Android-клиентом](android-client.md): mobile web — responsive SvelteKit, не PWA обязательно
