# Эволюция уведомлений

Не входит в v1. Черновик для следующего этапа после стабилизации базовой системы уведомлений.

## Цель

Сделать уведомления более «человечными» и предсказуемыми для долговых сценариев, не усложняя ежедневную настройку.

## Что уже есть в v1

- Каналы: Telegram, MAX (`a161`, `official`)
- Триггеры: долги, кредиты, плановые операции
- Локальное время отправки (`notification_time_local`) по timezone пользователя
- Разделение логики долгов:
  - `borrowed` (я должен)
  - `lent` (мне должны)
- Отдельные лимиты/задержки по просрочке

## Идеи после v1

- [ ] Тонкая политика частоты уведомлений:
  - «каждый день» / «по будням» / «раз в N дней»
  - подавление дубликатов по «тихим периодам» (cooldown)
- [ ] Раздельные шаблоны по направлению долга (`borrowed` vs `lent`) на уровне UI, а не только общей пары триггеров
- [ ] Режимы эскалации:
  - мягкое напоминание до срока
  - более заметный текст после X дней просрочки
- [ ] Гибкие окна отправки:
  - интервал времени (например, 09:00–21:00), а не только одна точка `HH:MM`
  - «не слать ночью» с учётом timezone пользователя
- [ ] Аналитика доставок:
  - счётчик отправок/ошибок по каналам
  - last successful send per trigger/channel
- [ ] Wizard в настройках:
  - пошаговая настройка «Я должен» / «Мне должны»
  - визуализация «когда именно придут сообщения»

## Открытые вопросы

- [ ] Нужны ли разные шаблоны для `debt_due_soon` и `debt_overdue` в разрезе направления долга?
- [ ] Нужен ли отдельный лимит уведомлений «в день срока» или достаточно текущей дедупликации?
- [ ] Стоит ли добавлять «тихие часы» глобально для всех триггеров?
