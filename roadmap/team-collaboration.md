# Командная / семейная работа

Планируется в **v1.5.0** ([ROADMAP](../ROADMAP.md#v150)).

## Зачем

Сейчас каждый `user_id` изолирован: свои счета, категории, операции. Семье или малому бизнесу нужен **общий учёт** с ролями.

## Модели (на выбор)

**A. Общее «домохозяйство» (предпочтительно для семьи)**

- Сущность household / workspace; члены с ролями.
- Счета, категории, операции принадлежат household; `created_by_user_id` — кто внёс.

**B. Расшаренные счета без полного слияния**

- Меньше миграции, сложнее статистика.

**C. Организация + сотрудники**

- Роли admin / accountant / viewer; audit log уже есть.

Для личной бухгалтерии чаще **A**.

## Что меняется

- Привязка данных к household + кто создал.
- Приглашения по ссылке / коду.
- Миграция: соло-пользователь → household из одного члена без поломки существующих установок.

## UI

- Участники, роли, исключение.
- В списке операций — кто создал.
- Переключатель нескольких household — можно отложить.

## Безопасность

- Нет доступа к чужому household.
- Удаление участника не удаляет историю.
- Админ инстанса ≠ admin household.
