# Кнопки в сообщениях Telegram / MAX

Не входит в v1. Черновик для обсуждения.

## Зачем

Сейчас ([этап 8](../stage_08_notifications.md)) уведомления — **текст** по шаблонам (долг, кредит, плановая операция). В сообщении с кнопками быстрее перейти в приложение или выполнить действие без копирования ссылки.

## Ограничение v1

Этап 8 — только **исходящие** сообщения, без приёма команд из Telegram/MAX. Кнопки нужно проектировать с учётом этого: не все типы кнопок возможны без inbound API.

## Варианты

| Тип | Telegram | MAX | Сложность |
|-----|----------|-----|-----------|
| **URL-кнопка** («Открыть в бухгалтере») | `InlineKeyboardButton` + `url` | зависит от API провайдера | низкая |
| **Callback** («Погасил», «Отложить») | `callback_data` + webhook/long polling бота | API a161 / official | высокая — нужен **входящий** канал |
| **Deep link** в URL | `/debts/{id}`, `/credits/{id}` с токеном или сессией | то же | средняя — безопасность ссылки |

**MVP после v1:** одна-две **URL-кнопки** в шаблоне — «Открыть долг», «Открыть кредит», «Все операции»; базовый URL из `system_settings.external_url` + путь.

## Шаблоны

Расширение [notification_templates](../stage_08_notifications.md) (этап 8):

- В UI редактора шаблона — опционально включаемые кнопки (не свободный JSON от пользователя — пресеты)
- Плейсхолдеры в URL: `{debt_id}`, `{credit_id}`, `{transaction_id}`
- i18n подписей кнопок: `notifications.buttons.open_debt`, …

## Telegram

- `sendMessage` + `reply_markup.inline_keyboard`
- Документация Bot API: inline keyboard
- Callback — отдельный roadmap-этап: endpoint для webhook, проверка `secret_token`, маппинг `callback_data` → действие в API

## MAX

- **a161** (`notify.a161.ru`) — уточнить в API, поддерживаются ли кнопки / ссылки в теле
- **official** (`platform-api2.max.ru`) — проверить формат вложений при send
- Единый интерфейс в `server/internal/notify/` — как для двух провайдеров текста

## Безопасность

- URL-кнопки не должны содержать API-токены в query (только публичные id сущностей + авторизация в браузере по сессии)
- При пустом `external_url` — кнопки не добавлять или текст «настройте внешний URL в админке»
- Callback-кнопки — подпись payload, TTL, привязка к `user_id`

## Этапы внедрения

1. URL-кнопки в 1–2 триггерах (просроченный долг, платёж по кредиту)
2. Настройка в редакторе шаблонов (вкл/выкл кнопки)
3. Callback / «погасил из чата» — только с inbound-каналом (отдельная задача)
