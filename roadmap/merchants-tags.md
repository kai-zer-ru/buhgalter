# Магазины (контрагенты), теги и комментарий

Планируется в **v1.5.0** ([ROADMAP](../ROADMAP.md#v150)).

Часть обогащения операций. Разбивка одной оплаты на несколько счетов — отдельно: [transaction-split-accounts.md](transaction-split-accounts.md) (v1.6.0). Вложения/фото к операции — вместе со [сканером чеков](receipt-scanner.md).

## Зачем

Сейчас у операции есть свободное **описание**. Нет нормализованного «магазина», меток и явного комментария — плохо для фильтров, статистики «куда уходит» и разбора чеков.

## Scope (v1.5.0)

| Поле | Суть |
|------|------|
| Магазин / контрагент | Нормализованное имя («Пятёрочка», «Яндекс»), справочник пользователя + подсказки из истории; фильтр и отчёты по магазину |
| Из чека | При скане QR/фото магазин подставляется из данных чека (если есть), пользователь подтверждает |
| Теги | Несколько меток на операцию («отпуск», «семья»); фильтр в списках и статистике |
| Комментарий | Отдельное поле заметки (или чёткая роль текущего `description`: кратко в списке / подробно в комментарии — решить в реализации) |

Пример: оплата наличными за автобус — магазин не обязателен; тег «транспорт» или правило категории (см. [category-rules-inbox.md](category-rules-inbox.md)). Покупка в магазине с чеком — магазин из чека + теги + комментарий «день рождения».

## MVP

1. Справочник магазинов (или строка с autocomplete → сохранение в справочник).
2. Поле на операции + фильтр в списке.
3. Теги many-to-many + фильтр `?tag=`.
4. Комментарий в форме операции (web + Android).
5. Сканер чеков передаёт предложенный магазин в форму.

## Не входит в этот пункт

- Разбивка суммы на несколько счетов/категорий ([transaction-split-accounts.md](transaction-split-accounts.md)).
- Полноценный CRM контрагентов.
- Автосоздание магазина из банка без подтверждения (можно предложить при разборе очереди правил).
