# Android-клиент

Не входит в v1. Черновик для обсуждения.

## Зачем

Веб-версия покрывает desktop и mobile browser, но нативное приложение даёт: иконку на экране, push-уведомления (когда появятся), биометрию, офлайн-просмотр последних данных, лучший UX на маленьком экране.

## Принципы

- **Тот же API** (`/api/v1/...`), авторизация по API-токену или сессии — без дублирования бизнес-логики на клиенте
- Сервер остаётся self-hosted; приложение — тонкий клиент к **вашему** инстансу (URL в настройках при первом запуске)
- Не конкурировать с вебом по полноте функций в первой версии клиента

## Технологии (на выбор)

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Kotlin + Jetpack Compose** | нативный UX, Material You | только Android |
| **Kotlin Multiplatform + Compose** | общая логика, потом iOS | сложнее старт |
| **Flutter / React Native** | один код на две платформы | тяжелее runtime, не «нативный» feel |

Склонение: **Kotlin + Compose**, если приоритет — только Android.

## MVP клиента (когда дойдём)

1. Настройка URL сервера, логин, сохранение токена (Android Keystore)
2. Главная: балансы, последние операции
3. Список операций, добавление расхода/дохода (упрощённая форма)
4. Счета — просмотр

Позже: переводы, долги, кредиты, push через FCM + серверные триггеры.

## Офлайн

- **v1 клиента:** только online; при обрыве — кэш последнего ответа read-only
- Полноценный offline-first (очередь операций) — отдельный большой этап, конфликты с сервером
